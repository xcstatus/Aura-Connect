## 一、产品定位与理念

### 1.1 产品概述

RustSSH 是一款现代化、高性能的 SSH 客户端，面向开发者和运维人员，提供类似浏览器的多窗口、多标签页管理体验。基于 Rust 构建，实现低资源占用和高响应速度。

### 1.2 核心理念

- **低资源占用**：空闲时 CPU < 1%，内存 < 50MB（单窗口），长时间运行无内存泄漏。
- **高性能**：终端渲染 ≥60fps，SSH 传输低延迟，支持大流量数据。
- **现代化设计**：扁平化界面、圆角元素、流畅动画、可定制主题。
- **安全优先**：凭据本地加密存储，支持系统级生物认证。
- **跨平台一致**：Windows/macOS/Linux 提供统一体验。

## 二、功能规格

### 2.1 窗口与标签页管理

#### 2.1.1 多窗口支持

- 支持创建多个独立窗口，每个窗口拥有独立的标签页栏、侧边栏和终端区域。
- 所有窗口共享全局配置（主题、快捷键、会话配置库）。
- 窗口关闭时自动保存布局状态。

#### 2.1.2 标签页管理

- **基本操作**：
  - 新建标签页（Ctrl+T）
  - 关闭标签页（Ctrl+W，或鼠标中键点击标签页）
  - 切换标签页（Ctrl+Tab 顺序切换，Ctrl+数字键跳转）
  - 标签页右键菜单：关闭、关闭其他、关闭右侧、重命名、复制标签页
- **标签页属性**：
  - 显示连接主机名或用户自定义标题
  - 连接状态指示器（绿色：已连接，黄色：重连中，灰色：未连接）
  - 未读输出指示（标签页标题高亮）
- **拖拽管理**：
  - 同一窗口内拖拽排序标签页
  - 跨窗口拖拽移动标签页
  - 拖拽标签页到空白区域创建新窗口
  - 拖拽窗口外部分离标签页为独立窗口

#### 2.1.3 窗口分割

- 支持水平分割（Ctrl+Shift+D）和垂直分割（Ctrl+Shift+E）
- 每个面板独立管理 SSH 会话和终端状态
- 面板支持拖拽调整大小、关闭面板
- 面板布局可保存为预设

#### 2.1.4 布局持久化

- 退出时自动保存所有窗口、标签页、面板的布局
- 启动时恢复上次的布局状态
- 支持布局快照（保存/加载自定义布局方案）

### 2.2 终端功能

#### 2.2.1 终端仿真

- **协议支持**：VT100、VT220、xterm 扩展
- **颜色支持**：16色、256色、True Color（24位）
- **字符支持**：完整 Unicode（包括中文、Emoji）、UTF-8
- **转义序列**：支持光标移动、清屏、颜色设置、窗口标题设置、剪贴板交互等标准序列
- **Alt 键处理**：支持 Alt 键作为修饰符（发送 Escape 前缀）

#### 2.2.2 文本交互

- **选择与复制**：
  - 鼠标左键选择文本（自动复制到剪贴板）
  - 双击选择单词，三击选择整行
  - 鼠标选择时按住 Alt 进行矩形选择
  - 复制快捷键：Ctrl+Shift+C（或 Ctrl+C 无选择时发送中断信号）
- **粘贴**：
  - 右键粘贴（或 Shift+Insert）
  - 粘贴快捷键：Ctrl+Shift+V
  - 支持多行粘贴确认（避免意外执行）
- **查找**：
  - Ctrl+F 打开查找面板
  - 支持正则表达式、大小写敏感、全词匹配
  - 高亮所有匹配项，支持上下导航

#### 2.2.3 鼠标事件

- **滚动**：鼠标滚轮滚动终端缓冲区，按住 Shift 滚动可发送箭头键
- **报告模式**：支持鼠标报告（应用程序可接收鼠标点击事件）
- **右键菜单**：复制、粘贴、清除滚动缓冲区、重置终端

#### 2.2.4 终端高级功能

- **滚动缓冲区**：可配置滚动行数（默认 10000 行）
- **响铃**：支持视觉响铃（窗口闪烁）和声音响铃
- **超链接**：识别 URL，Ctrl+点击可在浏览器打开
- **终端标题跟踪**：自动捕获并显示远程设置的终端标题

#### 2.2.5 字体与渲染

- **字体**：等宽字体优先，支持系统字体后备
- **渲染**：基于 GPU 的纹理缓存，仅增量更新
- **透明度**：支持终端背景半透明（可选模糊效果）
- **缩放**：Ctrl++/Ctrl+- 调整字体大小，支持缩放重置

### 2.3 会话管理

#### 2.3.1 会话配置库

- **存储策略**：遵循操作系统原生规范（OS Native Standards），确保存储位置的合规性与可发现性。
- **各平台路径映射**：

  | 平台 | 推荐存储路径 | 说明 |
  | :--- | :--- | :--- |
  | **macOS** | `~/Library/Application Support/rustssh/sessions.json` | 苹果官方推荐的应用支持数据存放点 |
  | **Windows** | `%AppData%\rustssh\sessions.json` | 用户漫游配置数据目录 |
  | **Linux** | `~/.config/rustssh/sessions.json` | 遵循 XDG Base Directory 规范 |

- **配置字段**：
  - 会话名称
  - 主机地址
  - 端口（默认 22）
  - 用户名
  - 认证方式（密码/私钥/代理）
  - 认证凭据引用（指向 Vault 存储）
  - 标签颜色
  - 启动命令（可选）
  - 分组标签
- **分组管理**：支持创建分组，拖拽整理会话
- **导入/导出**：支持从 PuTTY、OpenSSH 配置导入，支持导出为 JSON/YAML

#### 2.3.2 快速连接 (Quick Connect)

- **触发方式**：点击顶栏的快速连接图标（或使用快捷键启动弹窗/下拉面板）。
- **聚合功能**：
  - **命令直连**：输入连接命令，例如：`root@host:port` 或 `ssh://user@host:port`。
  - **最近连接**：展示最近使用的 SSH 会话历史（最多 50 条，可点击直连或清除）。
  - **会话搜索**：支持对已保存的会话配置库进行模糊匹配与实时搜索。
  - **新建入口**：面板底部提供“➕ 新建完整会话配置”入口，支持调起详细配置界面。
- **一键保存 (Quick Save)**：
  - 支持将临时输入的 `user@host` 格式地址一键保存至会话库，并自动生成 UUID 标识。
  - **模糊匹配规则**：
    - 匹配字段：会话名称 (Name) > 主机地址 (Host) > 用户名 (User) > 分组标签 (Tags)。
    - 权重：前缀匹配 (Prefix) 权重最高，其次是子串匹配，最后是拼音首字母简拼（可选扩展）。
  - **默认存储行为**：一键保存的会话默认存放在“未分类 (Uncategorized)”分组中，图标默认为 `Desktop`。

#### 2.3.3 新建/编辑会话配置规格 (Detailed Config)

当用户通过“新建完整会话配置”或编辑现有会话时，系统调起全屏蒙层面板（Modal Overlay），包含以下核心模块：

##### 1. 基础连接 (General)

- **会话名称**：必填，默认为 Host 地址。
- **主机地址 (Host)**：必填，支持 IP 或域名。
- **端口 (Port)**：默认为 22。
- **用户名 (User)**：必填。

##### 2. 身份验证 (Authentication)

- **认证方式**：下拉选择（Password / Private Key / SSH Agent / Interactive）。
- **凭据关联**：
  - 如果选择 Password，提供“存储到 Vault”复选框。若勾选，则加密并在 Vault 中创建关联 ID。
  - 如果选择 Private Key，提供文件选择器选择本地私钥路径，并支持输入私钥 Passphrase（可选存入 Vault）。

##### 3. 外观与组织 (Identity & Grouping)

- **标签颜色 (Color Tag)**：预设色卡（如：Red-生产, Green-开发, Blue-测试），对应 UI 上的圆角点或标签背景。
- **分组 (Folder)**：下拉选择已有文件夹或输入新文件夹名称。
- **备注 (Comments)**：多行文本框。

##### 4. 高级设置 (Advanced)

- **初始指令**：连接成功后自动执行的 Shell 命令。
- **代理跳转 (Proxy/Jump Host)**：支持关联另一个已保存的会话作为跳板机。

##### 交互细节

- **实时校验**：输入 Host 后自动进行格式校验。
- **连接测试**：面板底部提供“测试连接”按钮，在不保存配置的情况下尝试建立 SSH 握手并反馈结果。

#### 2.3.4 连接管理

- **断线重连**：自动检测连接中断，提示用户手动或自动重连（可配置重连次数和间隔）
- **连接超时**：可配置连接超时时间（默认 10 秒）
- **保活机制**：支持 TCP Keepalive 和 SSH 保活消息（可配置间隔）

### 2.4 认证与安全

#### 2.4.1 凭据存储架构

```
┌─────────────────────────────────────────────────────┐
│                    RustSSH App                      │
│  ┌─────────────────────────────────────────────┐   │
│  │            Credential Vault                  │   │
│  │  ┌─────────────────────────────────────┐    │   │
│  │  │  Encrypted Storage (AES-256-GCM)    │    │   │
│  │  │  - SSH passwords                     │    │   │
│  │  │  - Private key passphrases           │    │   │
│  │  │  - Custom authentication secrets     │    │   │
│  │  └─────────────────────────────────────┘    │   │
│  └─────────────────────────────────────────────┘   │
│                       │                            │
│                       ▼                            │
│  ┌─────────────────────────────────────────────┐   │
│  │         Platform Authentication             │   │
│  │  - macOS: Touch ID / Apple Keychain        │   │
│  │  - Windows: Windows Hello (PIN/Face/Finger)│   │
│  │  - Linux: libsecret (GNOME/KDE) / PAM      │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

#### 2.4.2 主密码与生物认证

- **首次启动**：提示设置主密码（用于加密 Vault）
- **解锁方式**：
  - 主密码输入
  - 生物识别（支持时）
    - macOS：Touch ID
    - Windows：Windows Hello（指纹、面部、PIN）
    - Linux：fprintd（指纹）、PAM
- **会话超时**：可配置 Vault 锁定时间（如 5 分钟无活动自动锁定）
- **手动锁定**：菜单栏或快捷键快速锁定 Vault

#### 2.4.3 认证方式

| 认证类型 | 说明 | 凭据存储 |
|---------|------|---------|
| **密码认证** | SSH 密码登录 | 加密存储在 Vault |
| **私钥认证** | RSA/Ed25519/ECDSA 私钥 | 私钥文件路径存储，密码存储在 Vault |
| **自定义认证** | 用户自定义脚本/程序认证 | 凭据模板，运行时调用 |
| **SSH Agent** | 转发本地 SSH Agent | 不存储凭据，直接转发 |
| **交互式认证** | 键盘交互（如二次验证） | 临时存储，会话结束清除 |

#### 2.4.4 Vault 安全增强规格

- **加密算法**：采用 AES-256-GCM 进行对称加密。
- **密钥派生**：使用 PBKDF2-HMAC-SHA256，默认迭代次数为 600,000 次，确保极高的抗爆破能力。
- **内存安全**：所有解密后的密钥和敏感情报在内存中均使用 `Zeroize` 协议，在对象释放时立即擦除物理内存，防止 Cold Boot 攻击。
- **重置策略 (Vault Reset)**：
  - 如果用户丢失主密码，系统支持“重置 Vault”操作。
  - **后果**：重置将永久删除所有已保存的凭据（密码、私钥密码），但**保留**会话配置列表（Host, User, Port 等非敏感信息）。
  - **流程**：用户需在设置界面手动确认重置，系统将删除旧的 `vault.json` 并引导创建新保险箱。

#### 2.4.4 主机密钥管理

- **存储位置**：`~/.config/rustssh/known_hosts`（与 OpenSSH 格式兼容）
- **首次连接**：
  - 显示主机指纹（MD5/SHA256）
  - 用户确认后添加到 known_hosts
- **密钥变更**：检测到主机密钥变化时，警告用户并阻止连接（用户可选择接受新密钥）
- **管理界面**：查看/删除已保存的主机密钥

#### 2.4.5 配置同步

- **同步方式**：
  - 支持自定义同步目录（如 iCloud Drive、Dropbox、Nextcloud）
  - 支持 WebDAV 协议同步
  - 支持自建同步服务器
- **同步内容**：
  - 会话配置（不含加密凭据，凭据不离开本地）
  - 主题配置
  - 快捷键配置
  - 布局预设
- **冲突解决**：自动合并，时间戳较新的优先

### 2.5 用户界面 (User Interface)

#### 2.5.1 主窗口布局 (Main Window Layout)

主界面采用极简的无边框合并顶栏设计，取消常驻侧边栏，充分放大终端活动区域。为了遵循各平台的交互契约，顶栏高度统一锁定为 **32px**。

**macOS 风格布局 (核心审美):**

- **布局顺序**：`[原生红绿灯] --- [滚动标签栏] --- | --- [操作组] --- [设置按钮]`
- **特性**：
  - 采用沉浸式透明标题栏（Transparent Titlebar）。
  - 顶栏高度固定 **32px**。
  - **红绿灯适配**：红绿灯区域宽度预留 **75px**，垂直居中对齐。在全屏模式下，红绿灯自动隐藏或靠边，标签栏宽度动态伸缩。

**Windows / Linux 风格布局:**

- **布局顺序**：`[滚动标签栏] --- [操作组] --- [设置按钮] --- [原生风格控制组]`
- **特性**：
  - 窗口操作按钮（最小化、最大化、关闭）位于最右侧，高度占满 32px。
  - 遵循平滑的线性图标规范（1px 线宽）。

#### 2.5.2 区域名词定义 (Terminology)

为了开发协作，定义以下核心术语：

| 术语 | 英文名 | 定义 |
| :--- | :--- | :--- |
| **顶栏** | Unified Header | 高度固定 32px，合并了窗口控件与应用交互入口的区域。 |
| **标签栏** | Tab Bar | 放置所有会话标签的容器，支持水平滚动。 |

#### 2.5.3 交互细节规范

- **标签排版**：标签标题文字统一**左对齐**，左边距 12px。
- **分隔符**：在标签栏与操作组之间存在 1px 宽度的垂直物理分隔线。
- **滚动逻辑**：采用水平滚动。当内容溢出时，右侧按钮保持固定，标签栏在蒙层下平滑滑动。
- **背景色选择**：采用 Apple Style Dark (`#1C1C1E`) 搭配顶栏稍浅色调 (`#28282D`)，形成层次感。

### 2.9 扩展与集成

#### 2.9.1 命令行接口

```bash
# 快速连接
rustssh user@host

# 指定会话连接
rustssh --session "Production Server"

# 打开新窗口
rustssh --new-window user@host

# 导入配置
rustssh --import-putty putty.reg

# 锁定 Vault
rustssh --lock
```

#### 2.9.2 IPC 通信机制与安全防范

- 允许外部程序向活动会话发送命令。
- 采用命名管道（Windows）或 Unix Socket（Linux/macOS）作为底层协议。
- **强制安全校验 (Peer Credential Validation)**：
  - 架构设计要求必须进行进程身份鉴别（验证连接 Socket/Pipe 的发起者是否为当前系统登录用户或同属主组的合法进程）。
  - 非法调用将被直接阻断，并在审计日志中报错。
- **高危操作防范**：在接受到 IPC 传来的敏感类 Shell 请求（如系统重启、批量删除、清空数据库命令）时，必须在 UI 界面弹窗强制进行人工确认 (Manual Confirmation) 才允许注入到终端流中。
- 支持的操作：安全发送文本、受限范围的命令行执行、唤起并切换标签页等。

---

## 三、 路线图与后续规划 (Future Works)

为了满足工控、嵌入式开发及老旧设备维护需求，RustSSH 将逐步扩展对非 SSH 协议的支持。

### 3.1 串口连接 (Serial Port) 支持

- **核心场景**：嵌入式设备调试、路由器/交换机 Console 口接入。
- **技术规格**：
  - **波特率 (Baud Rate)**：支持 9600, 115200 等标准及自定义速率。
  - **数据位/停止位/校验**：完整支持 8N1, 7E1 等配置。
  - **流控 (Flow Control)**：支持硬件 (RTS/CTS) 及软件 (XON/XOFF) 流控。
- **UI 适配**：
  - 自动探测系统可用串口（如 `/dev/tty.*` 或 `COMx`）。
  - 终端支持 HEX/ASCII 切换显示。
  - 状态栏实时显示串口 DTR/RTS 状态。

### 3.2 Telnet 协议支持

- **核心场景**：老旧网络设备管理、特定实验环境接入。
- **技术规格**：
  - 支持标准 Telnet (RFC 854) 及常见扩展选项。
  - **字符编码**：支持强制设置 GBK/UTF-8 编码，解决老旧设备乱码问题。
- **安全性提示**：在建立连接前，系统将通过 UI 显著位置提示“Telnet 为明文协议，不建议在不安全环境使用”。

### 3.3 架构可扩展性 (Protocol Agnostic)

为了支持上述规划，底层 `Session` 架构将进行抽象化改造：

- **Backend Trait**：定义统一的 `Read/Write/Resize` 接口。
- **SessionConfig 扩展**：
  - `protocol`: 增加 `SSH | Serial | Telnet` 枚举。
  - **按需显示字段**：在“新建会话”界面，根据所选协议动态切换配置面板（例如：选择 Serial 时隐藏 User/Password，显示串口号和波特率）。
