
## 三、技术架构

### 3.1 技术栈总览

| 组件 | 选型 | 说明 |
|------|------|------|
| **语言** | Rust 1.70+ | 内存安全，高性能 |
| **GUI 框架** | egui + eframe | 即时模式 GUI，支持无边框与跨原生窗口拖拽 |
| **终端引擎** | libghostty | C 库，核心通过 **Shared GPU Texture（共享纹理）** 映射给 egui 进行高性能贴图展示 |
| **SSH 协议** | russh | 纯 Rust 异步 SSH2 实现 |
| **异步运行时** | tokio | 多线程异步 I/O |
| **PTY 处理** | 原生 + tokio-pty | 跨平台 PTY 封装 |
| **加密** | aes-gcm, argon2 | Vault 加密 |
| **生物认证** | keyring-rs, security-framework, windows-rs | 跨平台系统认证集成 |
| **配置存储** | serde + serde_json | JSON 格式配置 |
| **同步** | reqwest + webdav | HTTP/WebDAV 客户端 |
| **日志** | log + tracing | 结构化日志 |

### 3.3 核心模块接口

#### 3.3.1 WindowManager

```rust
struct WindowManager {
    windows: HashMap<WindowId, WindowState>,
    global_state: Arc<RwLock<GlobalState>>,
}

impl WindowManager {
    fn create_window(&mut self, initial_tabs: Option<Vec<Tab>>) -> WindowId;
    fn close_window(&mut self, id: WindowId);
    
    // ==========================================
    // 核心难点攻坚：早期版本 MVP 必须实现的跨窗体交互
    // ==========================================
    // 将标签页跨越操作系统原生窗口级别（OS Native Viewport）进行拖拽交换
    fn move_tab(&mut self, tab_id: TabId, from: WindowId, to: WindowId, position: usize);
    // 从现有窗体剥离标签页，并动态实例化为新的操作系统原生窗口
    fn detach_tab_to_new_window(&mut self, tab_id: TabId, from: WindowId) -> WindowId;
    // 将独立窗口的内容吸附合并入统一窗体中
    fn merge_windows(&mut self, source_id: WindowId, dest_id: WindowId);

    fn list_windows(&self) -> Vec<WindowInfo>;
    fn focus_window(&self, id: WindowId);
}
```

#### 3.3.2 CryptoContext & Vault (内存态加密防护)

```rust
use secrecy::SecretString;
use zeroize::Zeroizing;

struct CredentialVault {
    // 禁止在明文中游荡对称密钥，使用安全句柄防护冷启动攻击
    master_key: Zeroizing<[u8; 32]>,
    unlocked: bool,
    unlock_timeout: Option<Duration>,
}

impl CredentialVault {
    // 强制使用 SecretString 隔离原始密码输入
    fn initialize(master_password: SecretString) -> Result<CryptoContext>;
    fn unlock(&mut self, credential: UnlockStrategy) -> Result<()>; 
    fn lock(&mut self);
    // 所有从系统取出的凭据，即使是明文也带有离开作用域后清空的标识
    fn get_credential(&self, key: &str) -> Option<Zeroizing<Vec<u8>>>;
    fn set_credential(&mut self, key: &str, value: &[u8]) -> Result<()>;
    fn delete_credential(&mut self, key: &str) -> Result<()>;
}
```

#### 3.3.4 协议连接与认证底座 (Protocol & Auth Backend)

```rust
/// 面向跨协议（SSH/Serial/Telnet）扩展的核心抽象
#[async_trait]
pub trait AsyncSession: Send + Sync {
    async fn read_stream(&mut self, buffer: &mut [u8]) -> Result<usize>;
    async fn write_stream(&mut self, data: &[u8]) -> Result<()>;
    async fn resize_pty(&mut self, cols: u16, rows: u16) -> Result<()>;
}

/// 策略模式的身份认证网关，解除协议绑定
#[async_trait]
pub trait Authenticator: Send + Sync {
    async fn authenticate(&self, backend: &mut dyn AsyncSession) -> Result<()>;
}
```

### 3.4 安全设计

| 安全层级 | 机制 |
|---------|------|
| **Vault 加密** | AES-256-GCM，密钥派生采用 Argon2id（针对暴力破解构建成本壁垒） |
| **内存防泄露** | 使用 `secrecy::SecretString` 隐藏口令，应用 `zeroize` 在内存作用域结束时自动清空明文密钥，防备冷启动与 RAM Dump 攻击 |
| **高亮沙盒边界** | 所有加解密行为只能通过全局唯一授予的 `CryptoContext` 句柄进行透明拦截调用，禁止业务层私配加密参数 |
| **传输安全** | SSH 协议自身加密，同步使用 HTTPS/WebDAV over TLS |
| **审计日志** | 所有 Vault 访问、敏感 IPC 调用记录跟踪 |

### 3.5 性能指标

| 指标 | 目标值 |
|------|--------|
| 启动时间（首次） | < 2 秒 |
| 空闲 CPU 占用 | < 1% |
| 空闲内存占用 | < 50MB（单窗口） |
| 终端渲染帧率 | ≥ 60fps |
| SSH 吞吐量 | 无显著降级（接近原生 SSH） |
| Vault 解锁延迟 | < 500ms（生物认证），< 100ms（主密码） |
