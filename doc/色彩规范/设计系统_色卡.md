# RustSSH 色卡设计规范 (Color System) v3.0

**版本**: 3.0
**适用范围**: 全应用 UI 主题、组件颜色、终端调色板
**设计目标**: 科技感、macOS 原生风格、高对比度可读性、支持多主题切换
**最后更新**: 2026-04-24
**更新内容**: 修正 Token 命名（`bg_surface`→`bg_secondary`、`text_muted`→`text_disabled`）；补充 Liquid Glass 专用 Token；修正方法名（`for_id`→`for_color_scheme`）。

---

## 一、设计原则

### 1.1 层级结构

```
Foundation Colors (基础色)
    ↓
Surface Colors (表面色)
    ↓
Semantic Tokens (语义色)
```

- **Foundation**: 最底层的 RGB 值，不可直接使用
- **Surface**: 基于 Foundation 的语义层级（`bg_primary`、`surface_1`、`surface_2`...）
- **Semantic**: 业务含义的颜色（`success`、`warning`、`accent`...）

### 1.2 主题分类

| 主题 ID | 名称 | 定位 | Accent 基色 |
|---------|------|------|-------------|
| `terminal_dark` | Terminal Dark | 默认主力主题 | #00FF80 (科技绿) |
| `terminal_light` | Terminal Light | macOS 浅色模式 | #007AFF (苹果蓝) |
| `nord` | Nord | 北欧蓝调 | #88C0D0 (冰川蓝) |
| `solarized` | Solarized | 护眼暗色 | #268BD2 (靛蓝) |
| `monokai` | Monokai | 经典程序员风格 | #66D9EF (天蓝) |
| `gruvbox` | Gruvbox | 暖色复古风格 | #FB4934 (暖红) |
| `dracula` | Dracula | 紫色主题 | #BD93F9 (紫罗兰) |

### 1.3 对比度要求

| 场景 | 最小对比度 | 示例 |
|------|-----------|------|
| 主文本 vs 背景 | 7:1 (AAA) | text_primary vs bg_primary |
| 次要文本 vs 背景 | 4.5:1 (AA) | text_secondary vs bg_primary |
| 大文本 (≥18px) | 3:1 (AA Large) | 标题文字 |
| 组件边框 vs 背景 | 3:1 | border_default vs bg_primary |

## 三、语义 Token 映射

### 3.1 背景层级 (Background Layers)

> **注**：Token 命名已与 `DesignTokens` 结构对齐。

```
bg_primary      ← 最底层页面背景
bg_secondary    ← 次级背景
bg_header       ← 顶栏/标签栏背景
bg_popup        ← 弹出层/模态框背景
surface_1       ← 卡片/输入框背景
surface_2       ← 悬停状态背景
surface_3       ← 按下/选中状态背景
tab_active_bg   ← 选中标签背景
```

### 3.2 文字层级 (Text Hierarchy)

```
text_primary     ← 主要信息、可读性最强
text_secondary   ← 辅助说明、次要信息
text_disabled    ← 禁用状态、占位符
```

### 3.3 边框层级 (Border Hierarchy)

```
border_default   ← 默认边框、分隔线
border_subtle    ← 更细的分隔、更弱的边界
```

### 3.4 状态色 (State Colors)

```
success          ← 成功、已连接
warning          ← 警告、处理中
error            ← 错误、失败
info             ← 信息、提示
```

### 3.5 Accent 色 (Accent Colors)

```
accent_base      ← 默认强调色
accent_hover     ← 悬停状态
accent_active    ← 按下/选中状态
on_accent_label  ← 按钮标签色（确保可读性）
```

### 3.6 Liquid Glass 专用 Token

这些 Token 用于玻璃材质效果，不可用于普通背景填充。

#### 暗色主题（基于白色渐变）

| Token | 暗色值 | 用途 |
|-------|--------|------|
| `glass_subtle` | rgba(255,255,255, 0.04) | 微弱存在提示 |

## 六、代码使用示例

### 6.1 Rust 引用方式

```rust
use crate::theme::{DesignTokens, tokens_for_state};
use crate::app::state::IcedState;

// ✅ 标准方式：从 state 获取 tokens（自动适配当前配色方案）
fn component(state: &IcedState) -> Element<'_, Message> {
    let tokens = tokens_for_state(state);
    // ...
}

// ✅ 通过配色方案 ID 获取 tokens（静态，不含用户自定义）
let tokens = DesignTokens::for_color_scheme("terminal_dark");

// ✅ 从 ColorScheme 转换（用于自定义配色流程）
let tokens = DesignTokens::from_scheme(&color_scheme);

// ✅ 在组件中使用
container.style(move |_| container::Style {
    background: Some(tokens.bg_primary.into()),
    border_color: tokens.border_default,
    ..Default::default()
})
```

> **注**：`for_color_scheme()` 在配色方案 ID 无效时自动回退到 `terminal_dark()`。若需更严格的错误处理，应使用 `ColorScheme::from_id()` 配合 `from_scheme()`。

### 6.2 CSS 变量映射（Web 导出）

```css
:root {
    --bg-primary: #1C1C1E;
    --bg-secondary: #232326;
    --text-primary: #FFFFFF;
    --text-secondary: #A0A0A5;
    --accent-base: #00FF80;
    --glass-subtle: rgba(255, 255, 255, 0.04);
    --glass-moderate: rgba(255, 255, 255, 0.08);
    /* ... */
}
```
