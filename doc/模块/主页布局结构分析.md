### 4.1 各层作用

| 层级 | 组件 | 作用 |
|------|------|------|
| 外层 container | padding 容器 | 控制内边距（20/20/16/16），使终端内容不贴边 |
| 内层 column | 间距容器 | 提供 10px 垂直间距，内部只有终端背景容器 |
| 终端背景容器 | Stack | 承载终端渲染层 + 覆盖层，纯黑背景 |
| Layer 1 | scrollable | 终端正文渲染区域，支持垂直滚动 |
| Layer 2 | scrollbar overlay | 自定义滚动条，跟随 hover 状态变化透明度 |
| Layer 3 | scrollback badge | "回滚" 提示标签，告知用户当前在查看历史 |

### 4.2 styled_terminal 渲染机制

`styled_terminal`（位于 `terminal_widget.rs`）负责终端行渲染：

- **缓存策略**：`RowWidgetCache` 按 `(row_index, generation)` 缓存每行片段
- **生成校验**：仅当 `generation` 发生变化时才重建该行 widget
- **性能优化**：大部分行命中缓存，只有光标行/选区行走完整渲染路径
- **光标处理**：使用 sin 驱动的 alpha 插值实现平滑闪烁（符合动画规范）
- **禁止动画**：终端渲染路径（styled_terminal）严格禁止添加任何动画逻辑

## 六、布局数据流（用于 PTY 尺寸计算）

```
┌─────────────────────┐
│   window_size       │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐    ┌──────────────────────────┐
│ TerminalViewportSpec │───▶│ terminal_scroll_area_rect │──▶ (x, y, w, h)
│  (包含 breadcrumb    │    └──────────────────────────┘
│   visible 状态)      │             │
└──────────┬──────────┘             ▼
           │            ┌──────────────────────────┐
           └───────────▶│ terminal_scroll_area_px   │──▶ (cols, rows)
                        └──────────────────────────┘
                                   │
                                   ▼
                        ┌──────────────────────────┐
                        │  PTY resize (cols, rows)  │
                        └──────────────────────────┘
```

`TerminalViewportSpec` 是窗口尺寸到 PTY 网格的单一数据源（SSOT），包含：

- `breadcrumb_visible` 控制 breadcrumb 区域是否计入 Y 坐标
- `top_bar_h`、`bottom_bar_h`、`main_column_gap` 控制垂直布局
- `terminal_panel_padding` 控制终端可滚动区域边界

## 七、已知问题

### 7.1 Breadcrumb 隐藏时终端区域上移而非变高 ✅ 已修复

**问题描述**：breadcrumb 隐藏时，`terminal_panel` 虽然变高了，但终端文本区域的最下一行没有下移——底部多出了空白，PTY 没有新增行。

**根因**：有两个独立问题叠加：

1. **breadcrumb 切换时没有触发 PTY resize**：`BreadcrumbTogglePin` / `BreadcrumbShowTemp` / `BreadcrumbHideTemp` 只更新了状态，没有调用 `apply_terminal_grid_resize_for_pane`，PTY grid 的 rows 数量不变。

2. **`spec.breadcrumb_visible` 从未被正确设置**：`terminal_viewport_spec_for_settings` 返回的 spec 中 `breadcrumb_visible` 永远是默认值 `true`，导致 `grid_from_window_size_with_spec` 计算的终端可用高度始终把 breadcrumb 区域算进去。即使触发了 resize，计算出的 rows 也是错的。

**修复**（`src/app/update/mod.rs`）：

- `apply_terminal_grid_resize_for_pane` 新增参数 `breadcrumb_visible: bool`，在计算 grid 前设置到 spec。
- `BreadcrumbTogglePin`、`BreadcrumbShowTemp`、`BreadcrumbHideTemp` 处理器中新增调用 `apply_terminal_grid_resize_for_pane`。
- `WindowResized` 和 `sync_terminal_grid_to_session` 同步修复，确保所有 resize 路径一致。

**效果**：breadcrumb 隐藏时 PTY grid 重新计算（rows 增加），PTY 向终端下发 resize，终端渲染新增行，底部空白消失。
