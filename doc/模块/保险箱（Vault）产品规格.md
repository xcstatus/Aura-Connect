## 一、模块概述

### 1.1 定位与职责

保险箱（Vault）是 RustSSH 的安全核心模块，负责集中管理用户的敏感凭据（SSH 密码、私钥口令）。通过主密码（Master Password）加密保护，确保即使磁盘数据被窃取，攻击者也无法获取明文凭据。

### 1.2 用户价值

- **隐私保护**：SSH 密码和私钥口令以强加密形式存储，防止本地数据泄露
- **统一管理**：所有敏感凭据集中在一个加密容器中，避免散落在配置文件各处
- **密码复用安全**：用户只需记住一个主密码，即可安全地管理多个 SSH 会话的凭据
- **凭据持久化**：凭据与会话配置分离，修改主密码不会丢失已保存的会话

### 1.3 核心概念

> 术语定义可查阅 `doc/名词定义.md`，以下列出 Vault 专属术语：


| 术语                            | 说明                                     |
| ----------------------------- | -------------------------------------- |
| **保险箱（Vault）**                | 存储 SSH 凭据的加密容器                         |
| **主密码（Master Password）**      | 用户设置的保险箱解锁口令，用于派生加密密钥                  |
| **VaultMeta**                 | 存储在 `settings.json` 中的保险箱元数据，包含盐值和验证哈希 |
| **加密盐（Salt）**                 | 随机数据，用于密钥派生，防止彩虹表攻击                    |
| **验证哈希（Verifier Hash）**       | PHC 格式的 Argon2id 哈希，用于验证主密码正确性         |
| **会话凭据（Session Credentials）** | SSH 密码和私钥口令的统称                         |

## 二、用户场景与使用流程

### 2.1 主要用户场景


| 场景编号  | 场景描述      | 触发条件                   | 用户目标                  |
| ----- | --------- | ---------------------- | --------------------- |
| SC-01 | 首次初始化保险箱  | 用户首次在安全设置中点击"设置保险箱"    | 创建一个加密容器，设置主密码        |
| SC-02 | 修改主密码     | 用户在安全设置中修改密码           | 更换主密码，同时迁移所有已有凭据      |
| SC-03 | 连接需要凭据的会话 | 用户点击已保存的会话发起连接         | 系统解锁保险箱，自动填充密码，无需手动输入 |
| SC-04 | 保存会话时保存密码 | 用户在会话编辑器中保存会话，勾选"记住密码" | 密码加密存入保险箱，下次连接自动使用    |
| SC-05 | 删除会话时清理凭据 | 用户删除已保存的会话             | 同步删除该会话对应的加密凭据        |
| SC-06 | 连接时保险箱已锁定 | 用户尝试连接已保存会话，但保险箱尚未解锁   | 弹出解锁界面，输入主密码后继续连接流程   |
| SC-07 | 保险箱尚未初始化  | 用户尝试保存密码但保险箱未创建        | 引导用户先在安全设置中初始化保险箱     |


### 2.2 核心用户流程

**流程 A：初始化保险箱**

1. 用户打开"设置中心" → "安全与凭据"
2. 点击"设置主密码"按钮
3. 在弹出的表单中输入新密码和确认密码
4. 点击"确认"，系统执行：
  - 生成随机 Salt
  - 通过 Argon2id 派生出验证哈希
  - 创建空的加密容器（Vault 文件）
  - 将 VaultMeta 写入 `settings.json`
5. 界面显示"保险箱已创建"，主密码自动存入内存解锁态

**流程 B：连接已保存会话（保险箱已解锁）**

1. 用户点击已保存的会话
2. 系统读取会话配置，发现有 `credential_id`
3. 从保险箱加载对应凭据（解密操作）
4. 自动填充密码，发起 SSH 连接

**流程 C：连接已保存会话（保险箱已锁定）**

1. 用户点击已保存的会话
2. 系统读取会话配置，发现有 `credential_id`
3. 弹出"解锁保险箱"界面（解锁弹窗）
4. 用户输入主密码
5. 后台执行密码验证 + 保险箱解密（KDF 耗时操作在后台线程执行，UI 不阻塞）
6. 验证通过后，自动填充密码，继续连接流程
7. 连接成功后，提示"是否将当前密码保存到保险箱"（可选）

**流程 D：修改主密码**

1. 用户打开"设置中心" → "安全与凭据"
2. 点击"修改主密码"
3. 输入旧密码、新密码、确认密码
4. 点击"确认"，系统执行：
  - 验证旧密码正确性
  - 从旧密码派生的密钥解密当前 vault
  - 用新密码派生新密钥，重新加密 vault
  - 更新 `settings.json` 中的 VaultMeta
5. 界面显示"密码已修改"

### 2.3 边界与异常


| 边界/异常            | 预期行为                  |
| ---------------- | --------------------- |
| 输入两次新密码不一致       | 提示"两次输入的新密码不一致"，不允许提交 |
| 旧密码错误            | 提示"旧密码校验失败"，不执行改密     |
| 保险箱文件不存在但元数据存在   | 显示"保险箱不可用"，引导用户重新初始化  |
| KDF 验证耗时期间用户关闭弹窗 | 后台任务继续执行，下次打开时重新验证    |
| 改密时 vault 文件缺失   | 提示错误，不执行任何操作          |

## 三、功能清单

### 3.1 功能矩阵


| 功能编号 | 功能名称         | 优先级 | 状态  | 备注                             |
| ---- | ------------ | --- | --- | ------------------------------ |
| F-01 | 保险箱初始化       | P0  | 已上线 | 设置中心安全页触发                      |
| F-02 | 主密码验证        | P0  | 已上线 | Argon2id KDF，后台线程执行            |
| F-03 | 保险箱解锁（UI 异步） | P0  | 已上线 | `spawn_blocking` 避免 UI 阻塞      |
| F-04 | 密码/口令保存      | P0  | 已上线 | 加密后存入 vault                    |
| F-05 | 密码/口令加载      | P0  | 已上线 | 解密后回填到连接表单                     |
| F-06 | 主密码修改（rekey） | P0  | 已上线 | 重新加密所有凭据                       |
| F-07 | 解锁弹窗路由       | P0  | 已上线 | pending_connect/delete/save 路由 |
| F-08 | 凭据同步删除       | P1  | 已上线 | 删除会话时同步删除凭据                    |
| F-09 | 连接后保存密码      | P1  | 已上线 | `pending_save_credentials` 流程  |
| F-10 | 自动锁定超时       | P1  | 规划中 | 尚未实现                           |
| F-11 | 后台切入锁定       | P1  | 规划中 | 尚未实现                           |
| F-12 | 生物识别解锁       | P2  | 规划中 | macOS Touch ID 等               |


### 3.2 详细功能说明

#### F-01：保险箱初始化

- **描述**：创建新的加密容器，生成 VaultMeta 并持久化到 `settings.json`
- **前置条件**：保险箱尚未初始化（`settings.security.vault` 为 `None`）
- **后置条件**：`settings.json` 中写入 `vault` 字段，`vault.json` 文件被创建
- **交互细节**：设置页触发，三步表单（旧密码可省略）
- **依赖项**：存储模块 `StorageManager`

#### F-04：密码/口令保存

- **描述**：将 SSH 密码和私钥口令加密后存入保险箱
- **前置条件**：保险箱已初始化，且运行时处于解锁态
- **后置条件**：`vault.json` 中追加/更新 `ssh:{session_id}` 凭据
- **交互细节**：会话编辑器"记住密码"开关触发，或连接成功后提示保存
- **依赖项**：CredentialVault 处于解锁态

#### F-06：主密码修改（rekey）

- **描述**：使用旧密码解密 vault，再用新密码和新 encryption_salt 重新加密（）
- **前置条件**：用户输入的旧密码验证通过，且 vault 文件存在
- **后置条件**：`vault.json` 完全重新加密（保留所有已有凭据），`settings.json` 中 VaultMeta 更新
- **交互细节**：改密期间 UI 显示加载状态，后台线程执行解密→重加密→写入
- **异常处理**：若 vault 文件不存在，提示"无法迁移，请重新初始化"

## 四、UI/交互规格

### 4.1 入口与布局

保险箱相关 UI 入口分布在两个位置：

1. **设置中心 → 安全与凭据**：保险箱初始化/修改入口
2. **解锁弹窗（VaultUnlockModal）**：保险箱锁定时连接会话的中间界面

**设置中心入口布局**：

- 位于"安全与凭据"页签
- 显示当前保险箱状态（未初始化 / 已锁定 / 已解锁）
- 按钮："设置主密码"（未初始化）或"修改主密码"（已初始化）
- 最后修改时间显示（审计信息）

**解锁弹窗布局**：

- 全屏遮罩 + 居中卡片
- 标题："请输入主密码以解锁保险箱"
- 密码输入框（显示/隐藏切换）
- 错误提示区域（密码错误时显示）
- 取消 / 解锁按钮

### 4.2 组件清单


| 组件               | 文件位置     | 状态       | 说明           |
| ---------------- | -------- | -------- | ------------ |
| VaultFlowModal   | 设置中心     | 初始化/改密流程 | 三步表单         |
| VaultUnlockModal | iced_app | 解锁流程     | 连接前凭据保护      |
| VaultStatusBadge | 设置中心     | 显示状态     | 未初始化/已锁定/已解锁 |


### 4.3 状态定义


| 状态名称                         | 触发条件                               | 视觉表现          |
| ---------------------------- | ---------------------------------- | ------------- |
| `VaultStatus::Uninitialized` | `settings.security.vault` 为 `None` | 显示"未初始化"及设置按钮 |
| `VaultStatus::Locked`        | 元数据存在但运行时未解锁                       | 显示"已锁定"及解锁提示  |
| `VaultStatus::Unlocked`      | 运行时 `vault_master_password` 有值     | 显示"已解锁"状态     |
| `VaultStatus::Unavailable`   | 元数据存在但 vault 文件不存在                 | 显示"不可用"警告     |

## 六、配置与数据

### 6.1 持久化数据


| 数据        | 存储位置                               | 格式   | 说明                       |
| --------- | ---------------------------------- | ---- | ------------------------ |
| VaultMeta | `settings.json` 的 `security.vault` | JSON | Salt、验证哈希、加密盐            |
| 凭据容器      | `vault.json`                       | JSON | Salt + 加密数据（AES-256-GCM） |


### 6.2 配置项


| 设置项                           | 类型                  | 默认值    | 说明               |
| ----------------------------- | ------------------- | ------ | ---------------- |
| `security.vault`              | `Option<VaultMeta>` | `None` | 保险箱元数据，存在时表示已初始化 |
| `security.vault_timeout_mins` | `u32`               | `5`    | 自动锁定超时分钟数（0=永不）  |
| `security.lock_on_background` | `bool`              | `true` | 切入后台时是否锁定        |

## 七、待办与规划

### 7.1 已知问题


| 问题              | 严重性 | 状态   | 备注                                                   |
| --------------- | --- | ---- | ---------------------------------------------------- |
| 无 vault 文件完整性验证 | 低   | 规划中  | 可考虑 HMAC 校验                                          |
| 内存中凭据数据无主动清理    | 中   | 部分修复 | 中间 buffer 的解密 payload 已用 Zeroizing 保护；凭据值本身（Vec）仍未零化 |
