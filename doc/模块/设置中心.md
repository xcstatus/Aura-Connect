## 一、 界面与交互规范

设置页面采用侧边栏导航的模块化布局。

### 1.1 布局规格 (Layout Specs)

为保证深层配置的可访问性，设置页面采用 **“一级侧边导航 + 二级顶部页签”** 的混合架构：

- **区域命名规范 (Terminology)**：
  - **设置导航栏 (Settings Sidebar)**：左侧 200px 区域，贯穿全高。
  - **设置标题栏 (Settings Header)**：右侧顶部 32px 区域，包含模块标题及**右侧关闭按钮**。
  - **模块分布区 (Module Tabs)**：内容区顶部的二级横向 Tab 栏。
  - **配置面板 (Configuration Panel)**：承载具体配置项的滚动区。
  - **底部提醒区 (Settings Bottom Banner)**：底部 50px 区域，用于显示“需重启生效”等实时操作提醒。
- **全局间距与衬距 (Paddings & Spacing)**：
  - **外部边距 (Margin)**：设置中心 Modal 距离主窗口边缘最小 **20px**。
  - **内部衬距 (Padding)**：
    - **设置导航栏**：左右衬距 **12px**，项间距 **4px**。
    - **配置面板**：统一使用 **24px** 环绕边距 (Symmetric Padding)。
    - **组件内间距**：配置项标题与描述之间为 **4px**，配置项之间为 **16px**。

- **整体结构 (Full-Height Sidebar Pattern)**：
  - **一级侧边栏 (Primary Sidebar)**：宽度固定为 **220px**，背景色为 `bg_secondary` (#232326)。**贯穿整个 Modal 高度（Sidebar to top）**，与内容区形成标准的左右分割。
  - **内容控制区 (Right Content)**：包含顶部的标题栏 (Header)、二级页签栏 (Tab Bar) 以及下方的配置面板 (Configuration Panel)。
  - **比例分配**：侧边栏固定 220px，右侧内容区自适应。
- **窗口尺寸联动 (Window Size Linking)**：
  - **显示模式**：设置页面以 **Modal Overlay (模态层)** 形式显示。
  - **尺寸比例**：
    - **宽度**：主页面宽度的 **85%**，但最小不低于 **800px**，最大限制为 **1200px**（避免大屏下行宽过长）。
    - **高度**：主页面高度的 **80%**，但最小不低于 **600px**。
  - **动态响应**：
    - 当主窗口缩小时，设置层按比例同步缩小，直至触碰最小尺寸阈值。
    - 触碰阈值后，设置层不再缩小，而是允许主页面出现半透明遮罩叠加，或者设置层自身保持开启位置居中。
- **二级页签栏 (Secondary Tab Bar)**：
  - 高度固定为 **40px**，底部带有 1px 的 `border_subtle` 分隔线。
  - **文字规则**：
    - 字号：**13px** (Semi-Bold)。
    - 间距：页签之间横向间距 **12px**。
  - **状态反馈**：
    - 选中态：文字使用 `accent_base`，并伴随底部 2px 高度的 `accent_base` 指示条。
    - 非选中态：文字使用 `text_secondary`，悬浮时切换至 `text_primary`。
- **表单布局 (Form Layout)**：
  - **配置项标题**：字号 **14px**，颜色 `text_primary`。
  - **配置项描述**：字号 **12px**，颜色 `text_secondary`。
  - **内部组件渲染**：
    - 组件（如 Switch/Dropdown）距离右侧边框衬距 **24px**（即配置面板的 Padding）。
    - 复杂输入框最大宽度 **480px**。

### 1.2 交互逻辑变更 (Interaction Upgrades)

1. **统一关闭方案 (Global Close Pattern)**：
   - 全局所有模态框统一使用右上角关闭按钮。
2. **即时保存与生效 (Instant Effect)**：
   - **移除保存与重置按钮**：配置项在修改后立即生效并同步至磁盘。
   - **交互反馈**：控件状态改变时，系统静默保存，无需用户额外点击。
3. **重启生效提醒 (Restart Notification)**：
   - **底部横幅**：针对需要重启才能生效的选项（如 GPU 加速切换），在设置中心**底部提醒区**显示醒目的警告色横幅。
   - **提示文案**：例如“⚠️ 部分设置需要重启应用后生效。 [立即重启] [稍后处理]”。
   - **视觉表现**：在主页面 **Unified Header 下方** 弹出一条醒目的黄色/橙色提示条（Banner）。
   - **文案内容**：`部分设置需要重启应用后生效。 [立即重启] [稍后处理]`。
   - **交互**：点击“立即重启”应调用进程重启逻辑。

### 1.3 设置中心原型图文字解析 (Dev Guide)

1. **视觉层次 (Visual Hierarchy)**：
   - 使用 `bg_secondary` (#232326) 作为侧边栏底色，与主内容区的 `bg_primary` (#1C1C1E) 形成明显对比。
   - 侧边栏与内容区之间使用 1px 的 `border_subtle` 垂直分割。
2. **交互行为 (Interactions)**：
   - **侧边栏切换**：点击左侧项（如终端配置），右侧内容区应立即切换，并默认选中该模块的第一个 Tab（如渲染引擎）。
3. **即时保存**：
   - 配置项修改后立即静默生效，无需点击按钮。

### 2.3 安全与凭据 (Security & Vault)

针对保险箱安全与主机指纹管理。

![安全与凭据详情 V2 - 中文规范版](file:///Users/xc/workspace/other/RustSsh/doc/模块/设计/security_vault_v2_zh.svg)
 
- **[模块] 安全与隐私 (Security & Privacy)**：
  - **保险箱安全 (Vault Security)**：
    - **初始引导 (Onboarding Flow)**：
      - **首次保存密钥引导**：当用户首次尝试保存 SSH 密钥、密码等敏感信息时，系统将检查保险箱（Vault）状态。
      - **密码强制设定**：若主密码尚未设置，系统将弹出全屏引导气泡或对话框，强制要求用户创建主密码，以确保后续持久化的安全性。
    - **自动锁定超时 (Auto-Lock Timeout)**：
      - 下拉选项：可选 1/5/10/30 分钟，或 **“永不”**。
      - 逻辑：选择时间后自动开启超时锁定；选择“永不”则保持保险箱处于解锁状态。
    - **切入后台时锁定 (Lock on Background)**：
      - 开关：当应用最小化或失去焦点切入系统后台时，立即锁定保险箱。
    - **主密码管理 (Master Password)**：支持强度检测，修改时需重现推导密钥（耗时约 1-2s）。
    - **系统生物识别 (System Biometrics)**：
      - 范围：支持 **Touch ID** / **Windows Hello** / **Linux Biometrics**。
      - 强制认证：开启或关闭此功能时，**必须**先通过一次系统生物识别认证，通过后方可切换状态。
      - 优先权：开启后，解锁保险箱时将优先唤起系统认证，失败后再降级为输入主密码。
  - **主机指纹管理 (Known Hosts)**：
    - 列表展示：查看已保存的主机 Key 指纹。
    - 快速清理：支持单选删除或全选重置。
  - **内存保护 (Privacy)**：
    - 资源回收：关闭设置界面时，自动执行敏感内存区域的零化 (Zeroize) 处理。
    - 列表展示：主机名 - 算法 (ed25519/rsa) - 指纹 (SHA256)。
    - 操作：支持批量勾选删除、手动添加受信指纹。
  - **连接策略**：
    - 下拉选择：
      - `Strict` (默认)：指纹不匹配或不存在时阻断并报警。
      - `Accept New`：自动接受新主机指纹。

### 2.3.1 安全模块原型图文字解析 (Security Specs)

针对安全模块（参考原型图 `rustssh_security_vaut_prototype`）：

1. **Vault (保险箱) 逻辑说明**：
   - **Auto-Lock (自动锁定)**：右侧为 `egui::Checkbox` 或 `Switch`。注意 `Idle timeout` 需联动，当超时开启时，下方下拉框才可交互。
   - **Master Password (主密码)**：
     - **强度条**：由 5 个矩形块组成，根据输入动态着色（Red/Yellow/Green）。
     - **上次变更时间**：使用小字号 `text_secondary` 置于强度条下方，作为审计参考。
2. **Biometrics (生物认证)**：
   - **指纹图标**：在开关右侧显示淡淡的指纹图标（IconContext），增强视觉暗示。
   - **平台差异**：在非 macOS/Windows 平台（如 Linux），此部分应自动隐藏或显示为“不支持”。
3. **视觉一致性**：
   - 分组标题（如 Auto-Lock）加粗并置于左上，描述文字紧随其下。
   - 所有交互控件（开关、下拉框）垂直居中于其所在行的右侧齐平。
