# 新增/编辑连接对话框 UI 规范（响应式版）

## 一、整体布局

### 1.1 对话框容器（响应式）
| 属性 | 值 |
|------|-----|
| 宽度 | 主页面宽度的 85% |
| 最小宽度 | 800px |
| 最大宽度 | 1200px |
| 高度 | 主页面高度的 80% |
| 最小高度 | 600px |
| 最大高度 | 无限制（内容超出时滚动） |
| 圆角 | 12px |
| 背景 | `Bg_Primary` |
| 阴影 | `shadow_float: 0 8px 24px rgba(0,0,0,0.4)` |

### 1.2 顶部区域
- **高度**：32px（留白区域）
- **关闭按钮**：位于右上角，距顶部 8px，距右侧 8px
  - 类型：Ghost 按钮
  - 尺寸：28px × 28px
  - 图标：X（16px × 16px）
  - Hover：背景 `Surface_1`
  - Active：背景 `Surface_2` + `scale(0.97)`

### 1.3 左右分栏结构（Flex 布局）
| 区域 | 宽度 | 背景 |
|------|------|------|
| 左侧导航栏 | 200px（固定） | `Bg_Secondary` |
| 右侧内容区 | 自适应（calc(100% - 200px)） | `Bg_Primary` |
| 左右间距 | 0 | 通过背景区分 |

**响应式行为**：
- 当对话框宽度 < 900px 时，左侧导航栏宽度保持 200px 不变
- 当对话框宽度 < 800px 时（触发最小宽度），整体保持 800px，内容可横向滚动（不推荐，优先保证主内容可见）

### 3.3 配置项区域（配置组）

每个配置组使用 `Surface_1` 背景（比主背景稍亮），圆角 10px，内边距 20px，组间间距 16px。

#### 3.3.1 通用（General）

| 字段 | 组件类型 | 必填 | 说明 |
|------|----------|------|------|
| 主机地址 | 输入框 | ✓ | placeholder: 192.168.1.100 或 example.com |
| 端口 | 数字输入框 | ✓ | 默认 22，范围 1–65535，步进 1 |
| 用户名 | 输入框 | ✓ | - |
| 认证方式 | 单选框组 | ✓ | 密码 / 私钥 |
| 密码 | 密码输入框 | 条件 | 选择“密码”时显示，带“显示/隐藏”切换 |
| 私钥路径 | 输入框 + 浏览按钮 | 条件 | 选择“私钥”时显示，浏览按钮为 Ghost 类型 |
| 密码短语 | 密码输入框 | 可选 | 选择“私钥”时显示 |

**布局**：每个表单项采用 Flex 布局，标签宽度 100px，控件自适应

#### 3.3.2 高级设置（Advanced）

| 字段 | 组件类型 | 默认值 | 说明 |
|------|----------|--------|------|
| 保持连接间隔 | 数字输入框 | 60 | 单位：秒，范围 0–3600 |
| 连接超时 | 数字输入框 | 10 | 单位：秒，范围 1–300 |
| 代理类型 | 下拉选择 | 无 | 无 / SOCKS5 / HTTP |
| 代理地址 | 输入框 | - | 选择代理后显示，placeholder: 127.0.0.1 |
| 代理端口 | 数字输入框 | - | 选择代理后显示，范围 1–65535 |
| 心跳包 | 开关 + 数字输入框 | 关闭 | 开启后显示间隔输入框（秒），默认 30 |

#### 3.3.3 端口转发（Port Forwarding）

> 采用平铺列表样式，每个转发规则前有圆形选择框（Radio，表示启用/禁用）

**本地转发**：
- 表头：状态 | 源端口 | 目标地址 | 目标端口 | 操作
- 每行：Radio（选中时生效） + 数字输入框（端口）+ 输入框（地址）+ 数字输入框（端口）+ 删除按钮（Ghost，X 图标）
- “+ 添加本地转发”按钮（Ghost，带加号图标）

**远程转发**：
- 结构同本地转发
- “+ 添加远程转发”按钮

**样式**：
- 表格无边框，行高 44px
- 输入框高度 32px（紧凑模式）
- Radio 尺寸 16px，与文字对齐

#### 3.3.4 加密方法（Encryption）

> 平铺列表样式，每个选项前有圆形选择框（Radio）

| 选择框 | 算法名称 | 说明 |
|--------|----------|------|
| ○ | aes256-gcm | 推荐，性能均衡 |
| ○ | aes128-gcm | 速度较快 |
| ○ | chacha20-poly1305 | 移动设备优化 |
| ○ | aes256-cbc | 兼容性优先 |

**附加选项**：
- 是否启用压缩：开关（默认关闭）
- HMAC 算法：下拉（sha2-256 / sha2-512 / 无）

### 3.4 底部按钮区
- **高度**：64px
- **对齐**：右对齐
- **按钮间距**：12px
- **上边框**：1px `Border.Subtle`（可选，增加视觉分隔）

| 按钮 | 类型 | 宽度 | 点击行为 |

## 四、焦点与交互反馈

### 4.1 Focus 状态
所有输入框、下拉框、按钮获得焦点时：
```
outline: 2px solid Accent.Base
outline-offset: 2px
```

### 4.2 Hover 状态（统一规则）
- 输入框/下拉框：边框变为 `Accent.Base`
- 按钮：按组件规范切换 Surface 层级
- 不叠加额外颜色，不产生发光效果

### 4.3 按下效果
```
transition: all 80ms cubic-bezier(0.4, 0, 0.2, 1)
transform: scale(0.97)
```
适用于所有可点击元素（按钮、选项、关闭按钮等）

### 4.4 键盘支持
| 按键 | 行为 |
|------|------|
| Enter | 保存（校验通过时） |
| Esc | 关闭对话框（取消操作） |
| Tab | 在可聚焦元素间循环 |
| ↑/↓ | 在左侧导航选项间切换 |

### 4.5 校验反馈
- 必填项未填：输入框边框变红（`State.Error`），下方显示红色提示文字
- 格式错误：同必填项反馈
- 端口范围错误：实时校验，提示“端口范围 1–65535”
- 保存时整体校验：滚动到第一个错误字段并聚焦

## 六、响应式行为总结

| 对话框宽度 | 布局表现 |
|------------|----------|
| ≥ 1200px | 宽度固定 1200px，居中显示 |
| 800px - 1200px | 宽度 = 主页面宽度的 85%，左右分栏正常 |
| < 800px | 触发最小宽度 800px，内容不缩放，整体可横向滚动（备选方案） |

| 对话框高度 | 布局表现 |
|------------|----------|
| ≥ 600px | 高度 = 主页面高度的 80%，内容区域滚动 |
| < 600px | 触发最小高度 600px，内容区域滚动 |
