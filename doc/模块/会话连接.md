# 快速连接入口：连接流程完整文档（含主机密钥认证与会话名称）

> 本文档定义从 快速连接页面中 点击会话或直连输入 `user@host[:port]` 到成功打开终端并显示 MOTD 的完整流程，包含：
> - 终端区域信息显示与清理规则
> - 连接成功时的执行时序与清屏逻辑（ED2 策略）
> - Vault 解锁弹窗、密码输入弹窗的打开/关闭时机
> - 主机密钥认证（首次接受 / 变更警告）
> - 有会话名称时优先显示会话名称
> - 错误处理与重试机制

---

## 1. 前置条件：Vault 状态

- Vault 默认可能处于 **锁定** 或 **解锁** 状态。
- 任何需要读取或保存凭据的操作（点击有凭据的会话、直连成功保存密码）都要求 Vault 已解锁。
- 若 Vault 锁定且操作需要访问凭据 → **先弹出 Vault 解锁弹窗**，解锁成功后再继续；用户取消则终止连接。

---

## 2. 通用连接流程（阶段划分）

以下流程适用于：
- 点击已保存会话（有/无凭据）
- 点击最近会话（等同于有凭据的会话）
- 直连输入 `user@host[:port]`

### 阶段 0：Vault 检查（按需）

| 场景 | 是否需要 Vault 解锁 |
|------|---------------------|
| 点击会话且已绑定 `credential_id` | ✅ 需要（读取密码） |
| 点击会话且无 `credential_id` | ❌ 不需要 |
| 直连输入（成功后保存密码） | 成功后需要（写入密码） |
| 直连输入（仅连接，未使用密码） | 不需要 |

若需要 Vault 且当前锁定：
- 弹出 Vault 解锁弹窗（标题：`解锁凭据库`，输入主密码）。
- 解锁成功 → 关闭弹窗，继续流程。
- 用户取消 → 终端显示错误：`[错误] Vault 未解锁，无法继续`，流程终止。

---

### 阶段 1：开始连接

**终端行为**：
- 完全清空终端区域（清除之前连接的 MOTD、错误信息等）。
- 显示连接目标信息：优先使用**会话名称**（若存在），否则显示 `user@host:port`。  
  例如：`正在连接 “我的开发机” (user@host:port) ...`

**动作**：
- 根据入口构建 Draft（直连解析 / 从会话加载配置）。
- 若直连输入缺 `user` → 弹出“补齐用户名”弹窗（非终端）。补齐后重新进入阶段 1。

---

### 阶段 2：认证准备

- 若会话已有 `credential_id` 且 Vault 已解锁 → 直接读取密码，跳过弹窗。
- 若会话无 `credential_id` 或直连需要密码 → 进入 **阶段 3 密码弹窗**（除非自动探测可免密）。

---

### 阶段 3：密码弹窗（交互式/密码兜底）

**打开时机**：
- 自动探测（Agent/Key）失败且服务器要求密码。
- 会话无 `credential_id` 且用户尚未输入密码。
- 用户手动选择“改用密码”。

**弹窗样式**：
- 模态框，标题：`输入密码 – <显示名称>`（优先会话名称，否则 `user@host`）。
- 输入框类型：密码
- 按钮：`连接`、`取消`

**关闭时机与后续**：
- 用户点击 `连接` → 关闭弹窗，终端显示：`正在认证...`，发起认证。
- 用户点击 `取消` → 关闭弹窗，终端显示：`连接已取消`，流程终止。

**密码错误重试**：
- 认证返回密码错误时：
  - 错误计数 +1。
  - 若错误计数 < 3 → 重新打开密码弹窗，显示剩余尝试次数（如`密码错误，还剩 2 次机会`）。
  - 若错误计数 = 3 → 关闭弹窗，终端显示：`[错误] 密码多次错误，连接已中断`，流程终止。**不保存任何凭据**。

---

### 阶段 4：自动探测授权提醒（仅首次）

- 触发条件：用户首次触发 Agent/Key 自动探测路径（可在连接前或连接中）。
- 弹窗内容：提示将使用系统 SSH Agent 或本地密钥，不会上传私钥。
- 按钮：`允许（本次）`、`始终允许`、`改用密码`。
- 选择 `改用密码` → 关闭弹窗，直接进入 **阶段 3 密码弹窗**。
- 选择 `允许` 或 `始终允许` → 关闭弹窗，继续自动探测。

> 授权选择持久化（`始终允许` 存设置）。

---

### 阶段 5：主机密钥认证

在 SSH 握手阶段，客户端会收到服务器的主机公钥。根据本地是否保存过该主机的密钥，分为三种情况：

#### 情况 A：首次连接（未知主机密钥）

- **终端显示**：  
  `正在验证主机密钥 ...`  
  然后暂停连接，弹出**主机密钥确认弹窗**。

- **弹窗内容**：  
  - 标题：`确认主机密钥`  
  - 显示：`服务器 <host:port> 的主机密钥指纹：`  
    `SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`  
  - 说明：`是否接受此密钥？以后连接将自动信任。`  
  - 按钮：`接受并继续`、`拒绝`、`查看指纹详情`（可选）

- **用户选择**：  
  - `接受并继续` → 将主机密钥保存到 `known_hosts`（或 Vault 中的信任存储），关闭弹窗，继续连接。  
  - `拒绝` → 关闭弹窗，终端显示：`[错误] 主机密钥验证失败，连接已终止`，流程终止。

#### 情况 B：主机密钥已保存且匹配

- 无弹窗，静默通过，终端可显示：`主机密钥验证通过`（可选，不强制）。

#### 情况 C：主机密钥已保存但不匹配（可能中间人攻击或服务器重装）

- **终端显示**：  
  `[警告] 主机密钥变更！`  
  然后暂停连接，弹出**主机密钥警告弹窗**。

- **弹窗内容**：  
  - 标题：`安全警告：主机密钥已变更`  
  - 显示：`服务器 <host:port> 的主机密钥与之前记录不匹配。`  
    `旧指纹：SHA256:old...`  
    `新指纹：SHA256:new...`  
  - 说明：`这可能意味着中间人攻击或服务器重新安装了系统。请谨慎操作。`  
  - 按钮：`接受新密钥（更新记录）`、`拒绝并断开`、`查看详情`

- **用户选择**：  
  - `接受新密钥` → 用新密钥覆盖旧记录，关闭弹窗，继续连接。  
  - `拒绝并断开` → 关闭弹窗，终端显示：`[错误] 主机密钥验证失败，连接已终止`，流程终止。

> 主机密钥存储位置：建议使用 Vault 或独立 `known_hosts` 文件（可配置策略）。本流程不强制具体存储实现，但要求上述交互。

---

### 阶段 6：连接过程中的错误处理

**任何错误（网络、协议、认证、超时、主机密钥拒绝等）都会：**
1. 终止当前连接尝试（停止底层 SSH 会话）。
2. 在终端区域显示明确的错误信息（见下表）。
3. 保留当前 Draft / 会话上下文（不自动清除）。
4. 在 Launcher 面板或终端上方提供操作按钮：`重试`、`编辑`、`切换认证方式`。

**错误类型与终端显示（示例）**

| 错误场景 | 终端显示 |
|---------|----------|
| DNS 解析失败 / 主机不可达 | `[错误] 无法连接到 <显示名称>，请检查地址和网络` |
| 认证失败（密码错误、密钥无效） | `[错误] 认证失败，请检查用户名或密码` |
| 主机密钥变更被用户拒绝 | `[错误] 主机密钥验证失败，连接已终止` |
| 连接超时 | `[错误] 连接超时，服务器无响应` |
| Vault 解锁失败/取消 | `[错误] Vault 未解锁，无法获取凭据` |
| SSH 协议错误（算法不匹配等） | `[错误] 协议协商失败` |
| 其他底层错误 | `[错误] 连接失败，请稍后重试` |

> 原则：不向用户暴露底层异常细节，只给出可操作提示。

---

### 阶段 7：连接成功

**终端行为**：
- **完全清空**之前的所有进度提示（“正在连接…”等）。
- 仅显示远程服务器返回的 MOTD（及随后的 Shell 输出）。

**保存动作（仅成功时执行）**：

| 入口 | 保存行为 |
|------|----------|
| 直连输入 | 按 `(user, host, port)` 去重：<br>- 若已存在相同键的会话 → 更新 `last_connected_at`，不创建新条目<br>- 若不存在 → 创建新会话，放入默认分组，**优先使用用户输入的 host 作为会话名**（可编辑）<br>- 若本次连接使用了密码 → 将密码写入 Vault 并绑定 `credential_id` |
| 无凭据会话 + 本次输入密码 | 将密码写入 Vault，更新会话的 `credential_id` |
| 有凭据会话（无需保存） | 无额外保存动作 |

- 保存失败（如 Vault 写错误）→ 终端显示警告：`[警告] 密码保存失败，请检查 Vault 状态`，但连接仍保持打开。

## 3. 终端信息清理与展示规则（汇总）

| 事件 | 终端行为 |
|------|----------|
| 新连接开始（阶段 1） | 完全清空旧内容（MOTD、错误、进度等） |
| 连接过程中（阶段 2~6） | 逐行显示当前阶段文本，可覆盖或追加，但不清除 |
| 发生错误（阶段 6） | 保留错误信息，停止进度更新 |
| 连接成功（阶段 7） | 完全清空所有本地进度信息，仅显示远程 MOTD + Shell |
| 用户点击“重试” | 重新从阶段 1 开始（再次清空终端） |

## 4. 弹窗总览（打开/关闭时机）

| 弹窗类型 | 打开时机 | 关闭时机 | 关闭后动作 |
|----------|----------|----------|-------------|
| Vault 解锁 | 需要读/写凭据且 Vault 锁定时 | 解锁成功 / 用户取消 | 成功→继续流程；取消→终止，终端显示错误 |
| 补齐用户名 | 直连输入缺少 `user` | 用户填写并确认 / 取消 | 确认→重建 Draft 并回到阶段 1；取消→终止 |
| 授权提醒 | 首次使用 Agent/Key 自动探测 | 用户选择任一按钮 | 根据选择继续或改用密码 |
| 密码输入 | 需要密码且无可用凭据 / 密码错误重试 | 点击“连接”或“取消” / 错误次数达 3 次 | 连接→发起认证；取消→终止；3 次错误→终止并显示错误 |
| 主机密钥确认（首次） | 首次连接未知主机 | 接受/拒绝 | 接受→保存密钥并继续；拒绝→终止 |
| 主机密钥警告（变更） | 已保存密钥与服务器不匹配 | 接受新密钥/拒绝 | 接受→更新密钥并继续；拒绝→终止 |

> 所有弹窗均为模态，同一时间最多显示一个。弹窗打开时连接流程暂停，关闭后继续。

---

## 5. 显示名称优先规则

在终端进度提示、弹窗标题、错误信息中，**若有会话名称（SessionProfile.name）**，则优先显示该名称，并可在括号内附上 `user@host:port`；若无会话名称，则仅显示 `user@host:port`。

**示例**：
- 会话名称为 `我的开发机`，地址 `root@192.168.1.100:22`  
  终端显示：`正在连接 “我的开发机” (root@192.168.1.100:22) ...`
- 直连输入无会话名称：  
  终端显示：`正在连接 user@example.com:2222 ...`

弹窗标题同理：`输入密码 – 我的开发机`

## 6. 完整状态跃迁示例（含主机密钥）

### 示例 1：首次连接直连，接受主机密钥

```
用户输入 user@host:22 并回车
  ↓
[阶段 0] Vault 无需解锁（无凭据）
  ↓
[阶段 1] 清空终端 → 显示“正在连接 user@host:22 ...”
  ↓
[阶段 2] 自动探测（Agent/Key）→ 首次使用 → 弹出授权提醒 → 用户允许
  ↓
[阶段 5] 主机密钥首次出现 → 弹出确认弹窗 → 用户接受
  ↓
认证成功（无需密码）
  ↓
[阶段 7] 清空终端 → 显示 MOTD → Shell 交互
  ↓
保存会话（无密码）
```

### 示例 2：主机密钥变更，用户拒绝

```
点击已保存会话“生产服务器”
  ↓
[阶段 0] Vault 解锁（读取密码）
  ↓
[阶段 1] 清空终端 → 显示“正在连接 “生产服务器” (root@10.0.0.1:22) ...”
  ↓
[阶段 5] 主机密钥不匹配 → 弹出警告弹窗 → 用户选择“拒绝并断开”
  ↓
终端显示“[错误] 主机密钥验证失败，连接已终止”
  ↓
流程终止，显示重试/编辑按钮
```

## 7. 连接成功时序与清屏逻辑

### 7.1 连接成功路径（三条代码路径）

SSH 连接成功后，有三条代码路径会触发"连接成功"状态，但行为完全一致：

| 函数 | 调用场景 | 文件位置 |
|------|---------|---------|
| `handle_connect_success()` | Agent/Key 自动认证成功 | `src/app/update/connection.rs` |
| `handle_connect_success_arc()` | 多路复用（连接池）成功 / 主路径 | `src/app/update/connection.rs` |
| 交互式认证成功 | `KeyboardInteractiveStep::Success` | `src/app/update/connection.rs` |
| 自动重连成功 | `handle_reconnect_tick` 重连成功 | `src/app/update/connection.rs` |

### 7.2 连接成功时的执行时序

以 `handle_connect_success_arc()` 为例，成功时的完整执行顺序如下：

```
1. attach_session_arc()       ← 将 SshChannel 挂载到 tab，建立 PTY 读写通道
2. sync_terminal_grid_to_session() ← 同步 PTY 尺寸到当前窗口大小
3. clear_local_preconnect_ui() ← ED2 (\x1b[2J\x1b[H) 清空整个屏幕缓冲区
4. pump_output(session)        ← 从 SSH 读取数据（通常含 MOTD），写入 VT grid
```

> 注：`quick_connect_flow = Connected` 的状态变更在第 1 步之前已完成。

**VT grid 行布局（清屏后）**：

```
清屏前（旧内容）：| [rustssh] 连接中… | SSH info | vault 提示 | 上次 MOTD | …
                         ↓ ED2 清空（整个缓冲区清零）
清屏后：
row 0: （空）
row 1: （空）
...
row N: （空）
                         ↓ pump_output 写入 SSH 数据
row 0: Last login: Sat Apr 11 ...   ← MOTD 第 1 行
row 1: dream@hinlink-h28k:~$       ← shell 提示符
row 2: _
```

### 7.3 清屏策略：ED2 vs 逐行清除

**当前策略**：统一使用 ED2（`\x1b[2J\x1b[H`）。

- **ED2**（erase display）：清空整个屏幕缓冲区（包括 scrollback 区域），光标回到 (0,0)
- **逐行清除**（已废弃）：只清除 `preconnect_info_line_count` 行数，存在行数计数不准导致残留问题

废弃逐行清除的原因：
- `preconnect_info_line_count` 只追踪 SSH info 行数，不包括 vault 提示行和 "连接中…" 行
- 多路复用路径跳过了 `inject_ssh_connecting_info`，导致计数为 0
- VT 渲染行为与预期不符（光标定位清除 vs 整屏清除效果不同）

### 7.4 关键数据结构

```rust
// 连接状态
pub enum QuickConnectFlow {
    Connected,   // 连接成功
    Connecting,  // 连接中
    Failed,      // 连接失败
    ...
}

// 连接阶段
pub enum ConnectionStage {
    VaultLoading,    // Vault 加载凭据
    SshConnecting,  // SSH 正在握手
    SessionSetup,   // 会话建立中
    ...
}

// 预连接信息行数追踪（已废弃，保留以兼容旧逻辑）
pub preconnect_info_line_count: usize, // SSH info 行数（3行）
pub vault_hint_line_count: usize,       // vault 提示行数（1行）
```

### 7.5 Vault 解锁流程的时序

Vault 解锁完成后，不在 vault.rs 中注入 SSH info，而是通过 `Message::ConnectPressed` 触发 `start_ssh_connect()` 自然注入：

```
vault.rs: handle_vault_unlock_complete()
  ↓ clear_local_preconnect_ui()         ED2 清屏
  ↓ inject "vault_unlocked"             vault_hint_line_count = 1
  ↓ dispatch Message::ConnectPressed     递归进入 update()

connection.rs: handle_connect()
  ↓ handle_connect() 入口：检测 Connected 状态则清屏（覆盖 vault 提示）
  ↓ start_ssh_connect()
      ↓ inject_ssh_connecting_info()   SSH info（3行）注入
      ↓ preconnect_info_line_count += vault_hint_line_count → 累加 vault 行数
      ↓ 发起异步 SSH 连接

... SSH 连接成功 ...

connection.rs: handle_connect_success_arc()
  ↓ clear_local_preconnect_ui()         ED2 清屏（清掉 vault + SSH info + "连接中…"）
  ↓ pump_output(session)               SSH 数据从 row 0 开始写入
```

## 8. 验收检查点（连接流程部分，含主机密钥）

- [ ] 点击有凭据会话：不弹任何框，直接连接成功，终端显示 MOTD。
- [ ] 点击无凭据会话：先弹出密码框 → 输入正确后连接成功 → 密码被保存。
- [ ] 直连 `user@host:port` 成功：自动生成/合并会话，终端清空后仅显示 MOTD。
- [ ] 密码错误连续 3 次：中断流程，终端显示错误，不再弹框。
- [ ] Vault 锁定时需要读取凭据：弹出解锁弹窗，取消则终端报错。
- [ ] **首次连接未知主机**：弹出主机密钥确认弹窗，接受后继续，拒绝则终止。
- [ ] **已保存主机密钥变更**：弹出警告弹窗，提供接受新密钥或拒绝的选项。
- [ ] **终端和弹窗标题优先显示会话名称**（若存在）。
- [ ] 任何连接错误：终端显示清晰错误信息，并提供重试/编辑按钮。
- [ ] 连接开始时终端完全清空，不会残留上次 MOTD 或错误。
- [ ] **连接成功后**：终端无 `[rustssh] 已连接。` 残留，MOTD 从 row 0 开始。
- [ ] **Vault 解锁后连接**：无 vault 提示行 + SSH info 残留。
