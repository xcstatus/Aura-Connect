# 终端配色方案

## 一、概述

终端配色方案功能允许用户自定义终端的颜色显示，支持预设方案选择和用户自定义配色。

### 1.1 功能特性

| 特性 | 描述 |
|------|------|
| 预设方案 | 7 种内置配色方案可选 |
| 终端颜色衍生 | 终端颜色自动从 UI 颜色衍生，可独立覆盖 |
| 用户配色 | 支持基于预设克隆、自定义修改 |
| 导入导出 | 支持 JSON 格式导入导出配色方案 |
| 实时预览 | 设置界面提供颜色预览 |
| 持久化 | 所有设置自动保存到配置文件 |

### 1.2 终端颜色衍生规则

终端颜色使用 `Option<Color>` 表示：
- `None` = 自动衍生自对应的 UI 颜色
- `Some(color)` = 使用自定义值覆盖衍生

| 终端颜色 | 衍生自 | 说明 |
|----------|--------|------|
| `term_bg` | `bg_primary` | 终端背景色 |
| `term_fg` | `text_primary` | 终端前景色 |
| `term_cursor` | `accent_base` | 终端光标色 |
| `term_cursor_bg` | `accent_base` | 块光标背景色 |
| `term_selection_bg` | `selection_bg` | 终端选中背景 |
| `term_selection_fg` | `text_primary` | 终端选中前景 |
| `term_scrollbar_bg` | `surface_1` | 终端滚动条背景 |
| `term_scrollbar_fg` | `surface_3` | 终端滚动条前景 |

## 二、数据结构

### 2.1 ColorScheme 结构体

定义在 `src/theme/tokens.rs`：

```rust
pub struct ColorScheme {
    // === UI 颜色 ===
    pub id: &'static str,
    pub name: &'static str,
    pub bg_primary: Color,
    pub text_primary: Color,
    pub accent_base: Color,
    // ... 其他 UI 颜色 ~20 个

    // === 终端颜色（可独立覆盖）===
    pub term_bg: Option<Color>,           // None = 衍生自 bg_primary
    pub term_fg: Option<Color>,           // None = 衍生自 text_primary
    pub term_cursor: Option<Color>,       // None = 衍生自 accent_base
    pub term_cursor_bg: Option<Color>,    // None = 衍生自 accent_base
    pub term_selection_bg: Option<Color>, // None = 衍生自 selection_bg
    pub term_selection_fg: Option<Color>, // None = 衍生自 text_primary
    pub term_scrollbar_bg: Option<Color>, // None = 衍生自 surface_1
    pub term_scrollbar_fg: Option<Color>, // None = 衍生自 surface_3

    // === ANSI 调色板 ===
    pub ansi: [Color; 16],
}
```

### 2.2 ColorScheme 衍生方法

```rust
impl ColorScheme {
    /// 获取生效的终端背景色
    pub fn effective_term_bg(&self) -> Color {
        self.term_bg.unwrap_or(self.bg_primary)
    }

    /// 获取生效的终端前景色
    pub fn effective_term_fg(&self) -> Color {
        self.term_fg.unwrap_or(self.text_primary)
    }

    // ... 其他 effective_* 方法
}
```

## 三、用户配色模块

### 3.1 UserColorScheme 结构体

定义在 `src/theme/user_scheme.rs`：

```rust
/// 用户自定义配色方案
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct UserColorScheme {
    /// 唯一 ID（格式: user_<timestamp>）
    pub id: String,
    /// 显示名称
    pub name: String,
    /// 基于的内置预设 ID
    pub based_on: String,
    /// 颜色覆盖映射表
    /// Key: 字段名（如 "bg_primary", "accent_base"）
    /// Value: HEX 字符串，None 表示重置为衍生值
    pub overrides: HashMap<String, Option<String>>,
    /// 创建时间
    pub created_at: i64,
    /// 修改时间
    pub modified_at: i64,
}
```

### 3.2 UserColorSchemes 列表

```rust
/// 用户配色方案列表

pub struct UserColorSchemes {
```

## 六、技术实现

### 6.1 配色应用流程

```
用户选择配色方案
        ↓
SettingsFieldChanged(ColorScheme)
        ↓
apply_settings_field()
        ↓
Settings::active_color_scheme()
        ↓
ColorScheme::effective_term_*() → 衍生值
        ↓
palette::scheme_vt_bytes() → VT 字节序列
        ↓
ghostty_vt::write_vt() → 发送到终端
        ↓
terminal.refresh_terminal_snapshots() → 刷新渲染
```
