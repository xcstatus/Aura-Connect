# RustSsh 动画使用规范 v1.0

> 本规范是 RustSsh UI 开发的核心规范之一，与 `doc/组件设计规范.md` 配套使用。

---

## 一、动画策略总览

### 1.1 核心目标

让用户体感 app 流畅。流畅感的来源是**每一步操作都有即时且自然的视觉反馈**，而非花哨的视觉效果。

### 1.2 动画分类

所有动画按目的分为两类：

| 类型 | 目的 | 手段 | 时长 |
|------|------|------|------|
| **感知反馈** | 让用户确信操作已被接收 | 即时视觉响应（缩放/颜色过渡） | 60-150ms |
| **状态指示** | 传达系统当前状态 | 进度/加载/切换指示 | 可变 |

**感知反馈**是流畅感的核心。终端用户操作频率极高，每一步交互（按键、点击、滚动）都应该在 **16ms 内** 开始视觉响应，在 **150ms 内** 完成整个过渡。

### 1.3 帧时间预算

应用使用动态帧率（`iced::time::every(tick_ms)`），tick_ms 根据性能在 16-100ms 之间浮动。动画必须在此约束下工作：

| 帧率 | tick_ms | 帧预算 |
|------|---------|--------|
| 60fps | ~16ms | 16.6ms/帧 |
| 30fps | ~33ms | 33.3ms/帧 |
| 10fps | ~100ms | 100ms/帧 |

动画插值必须基于 `tick_count` 和已知的 tick_ms 计算进度 `t ∈ [0, 1]`，**不能假设固定帧率**。

### 1.4 性能红线

```
终端渲染路径（terminal_rich.rs styled_terminal）严格禁止添加任何动画。
RowWidgetCache 的性能优化依赖于 generation-based cache hit。
任何改变样式属性的动画都会导致整个缓存失效。
```

**唯一例外**: 光标 alpha 插值（只影响单个光标样式，不破坏 cache）。

## 二、缓动函数规范

### 2.1 标准缓动函数库

所有缓动函数定义在 `src/theme/animation.rs` 中，接收 `t ∈ [0, 1]`，返回 `f32 ∈ [0, 1]`。

```
t=0 表示动画开始，t=1 表示动画结束。
```

#### ease_out — 快速启动，慢速收尾

适用于：按钮悬停进入、模态打开、颜色渐变进入。

```
ease_out(t) = 1 - (1 - t)^3
```

#### ease_in — 慢速启动，快速收尾

适用于：按钮按下释放、模态关闭退出。

```
ease_in(t) = t^3
```

#### ease_in_out — 两头慢中间快

适用于：Tab 切换、内容切换、状态变化。

```
ease_in_out(t) = t < 0.5 ? 4t^3 : 1 - (-2t + 2)^3 / 2
```

#### linear — 匀速

适用于：加载旋转、进度条、颜色透明度过渡。

```
linear(t) = t
```

### 2.2 缓动函数选用规则

| 动画场景 | 缓动函数 | 原因 |
|----------|----------|------|
| 按钮悬停进入 | `ease_out` | 快速到达悬停状态，有力 |
| 按钮按下下沉 | `ease_in` | 减速下沉，真实感 |
| 按钮悬停退出 | `ease_in` | 逆向自然 |
| Tab 切换 | `ease_in_out` | 平滑过渡 |
| 模态打开 | `ease_out` | 进入快 |
| 模态关闭 | `ease_in` | 退出从容 |
| 颜色/透明度过渡 | `ease_out` | 变化不应突兀 |
| 加载/进度指示 | `linear` | 匀速，指示时间准确 |

## 三、时长规范

### 3.1 按时长分级

所有动画时长必须从以下档位中选择：

| 档位 | 时长 | 适用场景 |
|------|------|----------|
| **即时** | 0ms | 纯状态变化，无过渡 |
| **快速** | 60-80ms | 按钮按下缩放 |
| **标准** | 100-150ms | 悬停过渡、Tab 切换 |
| **从容** | 200-300ms | 模态弹窗打开/关闭 |
| **信息** | >300ms | 仅用于进度指示，不用于交互反馈 |

**严禁超过 300ms 的交互反馈动画**。超过 300ms 会让用户感觉"迟钝"。

### 3.2 时长常量集中管理

所有动画时长常量**必须**定义在 `src/theme/layout.rs` 中，**禁止在组件代码中硬编码时长数字**。

```rust
// src/theme/layout.rs — 动画时长常量
pub const DURATION_HOVER_MS: u16 = 120;        // 悬停过渡
pub const DURATION_CLICK_MS: u16 = 80;         // 点击缩放
pub const DURATION_TAB_MS: u16 = 150;          // Tab 切换
pub const DURATION_MODAL_MS: u16 = 200;       // 模态打开
pub const DURATION_MODAL_CLOSE_MS: u16 = 150;  // 模态关闭
pub const DURATION_TAB_NEW_MS: u16 = 200;      // 新建 Tab
pub const DURATION_TAB_CLOSE_MS: u16 = 150;    // 关闭 Tab

// 缩放比例
pub const SCALE_CLICK: f32 = 0.97;             // 点击缩放比例
pub const SCALE_HOVER: f32 = 1.02;             // 悬停放大比例
```

## 四、组件动画规范

### 4.1 按钮系统

**文件**: `src/iced_app/widgets/chrome_button.rs`

#### 按钮状态机

```
[Idle] ──hover──→ [Hovered] ──press──→ [Pressed] ──release──→ [Hovered] ──exit──→ [Idle]
     ←────────────────────── exit ────────────────────────]
```

#### 动画规范

| 交互 | 动画属性 | 时长 | 缓动 | 备注 |
|------|----------|------|------|------|
| 悬停进入 | 背景色过渡 + 缩放至 1.02 | DURATION_HOVER_MS | ease_out | |
| 按下 | 缩放至 0.97 | DURATION_CLICK_MS | ease_in | |
| 悬停退出 | 逆向过渡至 Idle | DURATION_HOVER_MS | ease_in | |
| 禁用状态 | alpha 降至 0.45 | 0ms（即时） | — | 无过渡 |

#### 实现约束

- Iced 0.14 `button` widget 不直接支持 CSS transform 缩放。
- 通过返回不同的 `button::Style`（不同背景色、不同圆角视觉重量）实现悬停/按下的视觉区分。
- 颜色插值使用 `animation.rs` 中的 `lerp_color`。

### 4.2 Tab 标签系统

**文件**: `src/iced_app/view.rs` 中 `tabs_row` 相关代码

#### 动画规范

| 交互 | 动画属性 | 时长 | 缓动 |
|------|----------|------|------|
| 激活指示器滑动 | 下划线 X 位置插值 | DURATION_TAB_MS | ease_in_out |
| Tab 内容切换 | alpha 淡入淡出 | DURATION_TAB_MS / 2 | ease_in_out |
| 新建 Tab | 宽度 0→126px 展开 | DURATION_TAB_NEW_MS | ease_out |
| 关闭 Tab | 宽度 126→0px 收缩 | DURATION_TAB_CLOSE_MS | ease_in |
| 悬停关闭按钮 | 背景渐显 | DURATION_HOVER_MS | ease_out |

#### 实现约束

- Tab 动画状态存储在 `IcedState` 中。
- 每次 `TabSelected` message 时计算目标 Tab 的 X 偏移。
- 新建/关闭动画需要 `TabAnimState` 跟踪每个 Tab 的宽度。

### 4.3 模态弹窗系统

**文件**: `src/iced_app/view.rs` 中各 `*_modal_stack` 函数

#### 动画规范

| 阶段 | 动画属性 | 时长 | 缓动 |
|------|----------|------|------|
| 打开 — 遮罩 | alpha 0→0.42 | DURATION_MODAL_MS | ease_out |
| 打开 — 卡片 | translateY(-8→0) + alpha 0→1 | DURATION_MODAL_MS | ease_out |
| 关闭 — 遮罩 | alpha 0.42→0 | DURATION_MODAL_CLOSE_MS | ease_in |
| 关闭 — 卡片 | translateY(0→8) + alpha 1→0 | DURATION_MODAL_CLOSE_MS | ease_in |

#### 实现约束

- 遮罩透明度由 `modal_scrim_style` 函数驱动。
- 卡片 Y 偏移通过 `container` 的 `padding` 或 `Space` 高度实现。
- 动画状态存储在 `IcedState` 中，通过 `ModalAnimState` 跟踪。

### 4.4 终端区域

**文件**: `src/iced_app/terminal_rich.rs`

#### 零动画原则

终端正文区域（`styled_terminal` 函数）**严禁添加任何动画逻辑**。

原因：
1. 终端是 RustSsh 的核心功能区，渲染频率最高。
2. `RowWidgetCache` 的 4096 个行缓存条目依赖于 generation-based cache hit。
3. 每帧改变样式属性会导致整个缓存失效，造成严重性能问题。
4. 用户期望终端"即时响应"，任何动画都会造成输入延迟感。

#### 唯一例外：光标 alpha 插值

光标闪烁改为 `sin` 驱动的 alpha 而非跳变，不破坏 cache：

```rust
// 当前（生硬 — 每 32 帧跳变）
(tick_count / 32) % 2 == 0

// 改进（平滑 — sin 驱动，无跳变）
let phase = (tick_count as f32 * 0.2).sin();
(phase + 1.0) / 2.0  // 映射到 [0, 1]
```

#### 滚动条透明度

| 状态 | alpha | 过渡 |
|------|-------|------|
| 默认 | 0.08 | — |
| Hover | 0.25 | DURATION_HOVER_MS, ease_out |

通过 hover 状态跟踪，插值 alpha 值。

### 4.5 快速连接面板

**文件**: `src/iced_app/view.rs`

#### 动画规范

| 场景 | 处理方式 | 原因 |
|------|----------|------|
| 搜索结果过滤 | 无动画，即时响应 | 搜索是高频操作，延迟感会打断输入节奏 |
| 连接进度 dots | 3 帧循环（保持现状） | 进度指示无需更复杂动画 |
| 加载旋转符号 | 3 帧符号切换（保持现状） | Iced 限制下最优方案 |

## 五、帧插值引擎

### 5.1 核心函数

定义在 `src/theme/animation.rs`：

```rust
/// 线性插值。
pub fn lerp(a: f32, b: f32, t: f32) -> f32

/// 颜色插值。
pub fn lerp_color(from: iced::Color, to: iced::Color, t: f32) -> iced::Color

/// 根据 tick 计算动画进度 t ∈ [0, 1]。
/// enter_tick: 进入动画时的 tick_count
/// tick_ms: 每 tick 毫秒数
/// duration_ms: 动画总时长毫秒
pub fn anim_t(enter_tick: u64, tick_ms: f32, duration_ms: f32) -> f32
```

### 5.2 动画状态跟踪结构

每个需要动画的组件在 `IcedState` 中存储动画元数据：

```rust
// src/iced_app/state.rs

#[derive(Clone, Copy, Default)]
pub struct AnimState {
    pub enter_tick: u64,
    pub duration_ms: f32,
    pub from: f32,
    pub to: f32,
}

impl AnimState {
    pub fn new(from: f32, to: f32, duration_ms: f32, tick_count: u64) -> Self
    pub fn current(&self, tick_count: u64, tick_ms: f32) -> f32
    pub fn is_done(&self, tick_count: u64, tick_ms: f32) -> bool
}
```

## 六、性能注意事项

### 6.1 帧率降级策略

当应用进入低性能模式（`tick_ms` > 50ms）时：
- 动画时长按比例延长，但缓动函数不变。
- 即使动画变慢，缓动曲线形状不变。
- **禁止**: 不因性能问题跳过动画帧（会造成跳变感）。

### 6.2 禁止行为

| 禁止行为 | 影响 |
|----------|------|
| 在每帧重新分配 `Vec` | GC 压力 / 帧掉落 |
| 在动画计算中调用 `.to_string()` | 分配和字符串构建 |
| 对所有终端行应用动画 | RowWidgetCache 全部失效 |
| 动画期间重新构建整个 widget 树 | 大量 allocation |
| 硬编码时长数字 | 违反规范，难以统一调整 |

## 八、代码规范

1. **禁止在动画计算中创建新对象**: 所有插值使用 `f32`，颜色用 `iced::Color` 直接运算。
2. **禁止硬编码时长数字**: 使用 `layout.rs` 中的常量。
3. **禁止在终端渲染路径添加动画逻辑**: `styled_terminal` 和 `RowWidgetCache` 路径保持零动画。
4. **缓动函数必须命名导出**: 在 `theme/animation.rs` 中统一管理。
5. **动画状态必须有明确的起点和终点**: 不允许"无限循环"的动画（光标闪烁除外）。
6. **所有动画相关改动必须同步更新本文档**。
