# RustSsh 图标风格规范 (Icon Style Guide)

> **版本**：2.0
> **适用范围**：图标设计师、所有 UI 源码提交者
> **设计目标**：科技感、macOS 原生风格、极简线性、与组件规范无缝融合
> **更新说明 (2026-04-26)**：重新定义规范边界，专注于视觉/设计定义。代码实现细节移至 `doc/开发规范/UI开发规范.md`。

---

## 一、设计原则

1. **线性优先**  
   所有 UI 图标采用 **均匀描边** 风格，填充仅用于状态指示（如连接状态圆点）或强调性主图标。

2. **macOS 原生感**  
   遵循 SF Symbols 的设计语言：圆头端点、圆角转折、视觉平衡。避免尖锐的直角或复杂的内嵌细节。

3. **科技感表达**  
   通过几何断线、斜切角度、发光点缀（仅主图标）体现 "SSH/终端/网络" 的技术属性。

4. **状态驱动颜色**  
   图标颜色不单独定义，完全继承组件规范中的 `Text_Secondary`（默认）、`Text_Primary`（悬停/激活）、`Accent_Base`（选中/焦点）。

5. **响应式层级**  
   同一图标提供 16px、24px、32px 三种尺寸，线条粗细随尺寸微调，确保视觉重量一致。

## 二、技术规格（SVG 资源规范）

### 2.1 网格与画板

| 图标尺寸 | 画板大小 | 安全区 | 线条粗细 | 圆角半径（端点/转折） |
| :--- | :--- | :--- | :--- | :--- |
| 16×16 | 16px | 14px（留白 1px） | 1.0px | 1.0px |
| 24×24 | 24px | 22px（留白 1px） | 1.5px | 1.5px |
| 32×32 | 32px | 30px（留白 1px） | 2.0px | 2.0px |

- **留白规则**：图标主体不超出安全区，保证多尺寸对齐。
- **端点样式**：所有开放路径使用 **圆头端点**（Round Cap）。
- **连接样式**：转角使用 **圆角连接**（Round Join）。

### 2.2 线条与填充

- **默认 UI 图标**：纯描边，无填充（`fill="none"`）。
- **强调性图标**（如主图标、成功状态）：可填充 10%~20% 透明度的底色（遵循语义色）。
- **禁止**：渐变填充、多色描边、阴影或内发光（主图标除外）。

### 2.3 SVG 标准模板

```xml
<!-- 标准 SVG 模板（24×24 画板） -->
<svg width="24" height="24" viewBox="0 0 24 24" fill="none"
     xmlns="http://www.w3.org/2000/svg">
  <path d="..." stroke="currentColor" stroke-width="1.5"
        stroke-linecap="round" stroke-linejoin="round"/>
</svg>
```

| 属性 | 规范值 |
| :--- | :--- |
| `width` / `height` | 24 |
| `viewBox` | `0 0 24 24` |
| `fill` | `none` |
| `stroke` | `currentColor` |
| `stroke-width` | 1.5 |
| `stroke-linecap` | `round` |
| `stroke-linejoin` | `round` |

### 2.4 导出格式

- **矢量源文件**：SVG（`<svg>` 标签，无 CSS 依赖）。
- **命名规范**：`{category}_{name}.svg`，例如 `fn_plus.svg`、`action_download.svg`。
- **目录结构**：

```
assets/icon/
├── fn/          # 功能图标（新建、设置、关闭等）
├── nav/         # 导航图标（箭头、折叠展开等）
├── action/      # 动作图标（下载、上传、删除等）
├── status/      # 状态图标（连接、错误、加载中等）
├── file/        # 文件图标（文件夹、文档等）
└── app/        # 应用图标（SFTP、锁、解锁等）
```

## 三、图标分类与样式

### 3.1 应用主图标 (App Icon)

- **设计元素**：终端提示符 `$` 与盾牌（安全）结合，嵌入闪电（速度）斜切线条。
- **样式**：
  - 填充渐变：深色背景（`#1C1C1E` → `#000000`）
  - 主图形：白色描边（2.5px 外描边），内部断线使用 `Accent_Base`（暗色主题下为 `#00FF80`）
- **尺寸**：提供 512×512、256×256、128×128、64×64、32×32、16×16。
- **圆角**：macOS 风格大圆角（半径 = 边长 × 0.225）。

> **示例概念**：一个带圆角的黑色底板，中央是 `>_` 符号，其中 `_` 下划线向右上倾斜 12°，并延伸为闪电尾部。

### 3.2 模块/工具栏图标 (Module & Toolbar Icons)

用于顶栏、设置面板、新建会话界面等。遵循"线性、均匀、简洁"。

| 图标名称 | 语义 | 关键形状 | 特殊约束 |
| :--- | :--- | :--- | :--- |
| 新建 (plus) | 新建标签/会话 | 十字交叉，两臂等长 | 不允许使用加粗变体 |
| 快速连接 (bolt) | 快速连接弹窗 | 闪电符号，底部斜切 | 与主图标的闪电方向一致 |
| 设置 (gear) | 打开设置 | 六角齿轮，中心圆点 | 齿数为 6，齿顶圆角 |
| 查找 (magnifyingglass) | 查找面板 | 放大镜，手柄与镜身夹角 45° | 手柄长度 = 镜身直径 × 0.8 |
| 分割水平 (split_h) | 水平分割窗口 | 一条水平线，上下带三角箭头 | 箭头仅为指示，不闭合 |
| 分割垂直 (split_v) | 垂直分割窗口 | 一条垂直线，左右带三角箭头 | 同上 |
| 关闭 (xmark) | 关闭标签/面板 | 两条对角线交叉 | 线宽等于标准线宽，无圆环 |
| 复制 (copy) | 复制文本 | 两个重叠矩形，偏移 (2,2)px | 前层实心，后层描边 |
| 粘贴 (clipboard) | 粘贴 | 剪贴板 + 右上角下折角 | 折角使用 45° 斜线 |
| 锁定 (lock) | 锁定 Vault | 锁体 + 锁梁 | 锁梁为半圆弧，锁体圆角矩形 |
| 解锁 (lock_open) | Vault 已解锁 | 同上，但锁梁打开 | 锁梁与锁体分离 1px |

**颜色映射**（由组件规范驱动）：

| 状态 | 颜色 Token |
| :--- | :--- |
| 默认 | `Text_Secondary` |
| 悬停 | `Text_Primary` |
| 激活/选中 | `Accent_Base` |
| 禁用 | `Text_Disabled` |

### 3.3 状态指示图标 (Status Icons)

位于标签页、侧边栏、状态栏，表示连接状态、通知、错误等。**允许使用填充 + 描边混合**。

| 状态 | 形状 | 颜色（语义色） | 尺寸 | 动效 |
| :--- | :--- | :--- | :--- | :--- |
| 已连接 | 实心圆 | `Success` | 8×8 | 无 |
| 连接中 | 空心圆 + 圆弧旋转 | `Warning` | 10×10 | 匀速旋转（1s/圈） |
| 未连接 | 实心圆 | `Disabled`（灰） | 8×8 | 无 |
| 重连中 | 两个半圆弧交错 | `Warning` | 10×10 | 双向旋转 |
| 错误/断开 | 圆内嵌感叹号 | `Error` | 12×12 | 无 |
| 有未读输出 | 圆点（右上角标记） | `Accent_Base` | 6×6 | 呼吸效果（alpha 0.5→1） |
| Vault 锁定 | 锁（填充） | `Text_Secondary` | 12×12 | 无 |
| Vault 解锁 | 开锁（填充） | `Accent_Base` | 12×12 | 无 |

**状态栏布局**：状态图标与文字组合，图标居左，间距 4px。

### 3.4 侧边栏分组/操作图标

| 名称 | 形状 | 尺寸 | 行为 |
| :--- | :--- | :--- | :--- |
| 展开箭头 (chevron_right) | 右向 V 形 | 12×12 | 旋转 90° 表示展开 |
| 折叠箭头 (chevron_down) | 下向 V 形 | 12×12 | 旋转 -90° 表示折叠 |
| 分组文件夹 (folder) | 文件夹轮廓 | 16×16 | 默认 `Text_Secondary` |
| 会话设备 (desktop) | 显示器屏幕 | 16×16 | 默认 `Text_Secondary` |
| 快捷方式 (link) | 链环 | 14×14 | 右下角小箭头 |

**侧边栏图标规则**：

| 状态 | 颜色 Token |
| :--- | :--- |
| 默认 | `Text_Secondary` |
| 悬停 | `Text_Primary` |
| 选中 | `Accent_Base` |

## 四、颜色使用细则

### 4.1 主题适配

所有图标颜色通过 CSS 变量或主题 Token 动态绑定，**禁止硬编码色值**。颜色映射由 `IconState` 枚举定义，详见 `doc/开发规范/UI开发规范.md` 中的"图标状态与颜色映射"章节。

### 4.2 特殊场景

- **主图标**：独立于主题，使用深色底板 + 白色/Accent 描边。
- **错误/警告状态**：直接使用 `State_Error` / `State_Warning`，不经过主题映射。
- **蒙层/弹出层**：图标颜色跟随所在 Surface 的文本颜色（例如 `Bg_Popup` 上使用 `Text_Primary`）。

## 五、布局与对齐

### 5.1 图标与文字间距

| 场景 | 间距（图标右边缘到文字左边缘） |
| :--- | :--- |
| 按钮内（带文字） | 6px |
| 标签页标题前 | 6px |
| 侧边栏条目 | 8px |
| 状态栏 | 4px |

### 5.2 图标在按钮中的位置

- **仅图标按钮**：图标居中于按钮区域（Padding 内部已包含）。
- **图标+文字按钮**：图标左对齐，文字紧接其后（不可将图标置于右侧）。

### 5.3 基线对齐

图标与文字使用 **垂直居中对齐**（`align-items: center`）。对于特殊尺寸（如 12px 图标与 13px 文字），手动微调 `dy` 使视觉平衡。

## 六、动效与反馈

### 6.1 悬停

- 仅改变颜色（`Text_Secondary` → `Text_Primary`），不缩放、不移位。
- 过渡时间：`120ms`（符合组件动效规范）。

### 6.2 点击

- 对于 **可点击的图标按钮**（如关闭标签页的 ×）：缩放 `0.97`，持续时间 `80ms`。
- 对于 **状态图标**（如连接指示器）：无点击反馈。

### 6.3 加载/等待

- 连接中/重连中图标：匀速旋转，使用 `transform: rotate` 动画。
- 旋转中心：图标几何中心。

## 七、禁止清单

- ❌ 使用非均匀线宽（如笔画末端变细）。
- ❌ 图标内部使用多于两种颜色（除主图标外）。
- ❌ 为图标添加投影、内发光或描边模糊。
- ❌ 在终端渲染区域内使用任何图标动画（光标闪烁除外）。
- ❌ 使用位图（PNG/JPG）作为 UI 图标（仅允许 SVG 或字体图标）。
- ❌ SVG 中硬编码颜色值（必须使用 `currentColor`）。
