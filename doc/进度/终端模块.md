## 终端模块现状梳理（已完成 / 待完成 / 待优化）

## 里程碑

### P2.1 启动（2026-04-01）

- **Gate A（功能正确性）**：
  - styled 渲染主路径已稳定（Iced styled + 光标/宽字符/选择复制/滚动闭环已具备）
  - key/focus/paste 已统一收敛到桥接层（`TerminalHost`）与终端核心（`terminal_core`）
- **Gate B（性能基线）**：
  - 已产出 idle / throughput / longrun 三段数据与 meta（复现上下文齐全）
  - 行级 dirty 增量路径已落地（避免常态全量重建）
- **Gate C（可回滚性）**：
  - 渲染路径可回退（Plain/Styled 可切换）
  - 回归脚本可执行（键盘矩阵）

### 已完成（能用、且有基本工程化保障）
- **数据链路打通（SSH → UI → VT → 渲染）**：`TerminalView` + `TerminalSessionBridge` + `GhosttyVtBackend` + `VtTerminalWidget`
- **每帧限额 drain（避免 UI 卡死/高 CPU）**：`src/terminal/session_bridge.rs`（byte/read budgets + pending remainder）
- **PTY resize 同步（带去抖）**：`TerminalView` 计算 cols/rows，`bridge.resize_pty_if_needed`
- **终端渲染（更像真实终端）**
  - **颜色/背景色/粗体/下划线/inverse**：基于 libghostty render-state（`ghostty_render_state_row_cells_get(...STYLE/FG/BG...)`）
  - **按行 dirty 增量更新缓存**：`GhosttyVtTerminal::update_dirty_styled_rows_and_clear_dirty` + `VtTerminalWidget` 的 `cached_rows`
- **滚动（优先用 libghostty 能力）**：鼠标滚轮 → `ghostty_terminal_scroll_viewport(DELTA)`（`scroll_viewport_delta_rows`）
- **关键单测覆盖（GhosttyVt wrapper / bridge）**
  - `GhosttyVtTerminal`：new/write/snapshot grid/resize/encode_key/dirty 行更新
  - `TerminalSessionBridge`：budget、debounce
- **“无显示”问题复盘文档已沉淀**：`doc/终端区域无显示问题复盘.md`
- **Iced styled 主路径：网格与鼠标命中对齐**：视口与列带由 `src/iced_app/terminal_viewport.rs`（含 `terminal_scroll_cell_geometry`）统一推导；`src/iced_app/terminal_rich.rs` 用每格 `Fixed(cell_w)` + `Row` 与命中测试同源，避免纵/横向拖选与点击错位。排查记录见 `doc/终端网格命中测试排查复盘.md`。

### 待完成（功能缺口，会直接影响“像终端”）
- **光标渲染（持续收敛）**：Iced 路径已支持块/下划线/竖线与闪烁；与全场景像素级一致、以及非 Iced 渲染路径上的行为仍可能对齐官网行为继续收紧。
- **选择/复制（多路径）**：Iced styled 主路径已按 PTY 列网格做拖选与命中；其它代码路径（legacy widget、极端宽字符格）若仍有偏差，按 `doc/终端网格命中测试排查复盘.md` 逐项收敛。
- **粘贴（含 bracketed paste/安全校验）**：libghostty 有 paste utilities（可优先接入）
- **Tab/窗口焦点事件**：focus in/out 编码（libghostty 有 focus encoding）
- **更完整的键盘协议**：目前 backend 侧 `on_key` 还是 minimal fallback；理想是复用 `GhosttyVtTerminal::encode_key` 覆盖更多键（含修饰键、功能键）

### 待优化（现在能跑，但体验/性能/正确性还有明显提升空间）
- **字体度量与对齐**：**Iced 主路径**已使用 `terminal_viewport` 的字体度量与 `cell_w = w_map / cols`，与 PTY resize 同源。历史上或 **非 Iced** 路径里仍有 `char_w=8.5` 等常量近似、宽字符与不同字体 advance 偏离网格等问题时，需向 `terminal_viewport` 口径统一。
- **宽字符/组合字符**：当前 run 宽度用 `chars().count()`，对 emoji/中日韩宽字符不准
- **滚动条 UI**：libghostty 提供 `GhosttyTerminalScrollbar`（total/offset/len），目前未画 scrollbar，也没显示“是否在回滚”
- **渲染性能进一步优化**
  - 只在可见区域绘制（当前按 cached_rows 画到 rect 底部已做，但 run 级别仍可优化）
  - 更低开销的粗体实现（目前是“双绘制”近似）
- **Debug HUD 清理/开关化**：`TERM-RECT`/`VT-PAINT` 等仅 debug 下存在，建议集中成可开关 feature，避免长期噪声
- **结构一致性**：`src/terminal/mod.rs` 里的 `TerminalEngine/Renderer` trait 目前与实际实现（backend/widget）还未完全对齐（后续可统一边界）

如果你希望我把这些条目变成一个“按优先级 + 对应文件/接口 + 预估工作量”的执行清单，我可以直接按“用户可感知收益最大”的顺序排下来。
