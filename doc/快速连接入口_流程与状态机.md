# 快速连接入口：流程与状态机（开发指导）

本文档定义“快速连接入口（Launcher）”的**页面结构、流程分支、状态机、数据持久化与验收标准**，用于指导后续开发与优化。

范围：

- 快速连接面板（搜索/直连一体）
- 从最近/已保存会话打开连接
- 认证自动探测（Agent/Key）与交互式/密码兜底
- 直连成功后“强一致保存”（策略 A）与去重
- 密码错误多次中断流程

非范围（后续文档补充或引用已有文档）：

- 主机密钥策略（`HostKeyPolicy`）的具体落地（见 `SSH连接与凭据安全链路分析.md`）
- russh 算法集策略（KEX/Cipher/MAC/Compression）
- 多协议（非 SSH）连接入口

---

## 术语与对象模型

### 1) 连接草稿（Draft）

用户在输入框直连时生成的临时对象，用于承载“尚未保存但可发起连接”的信息。

最小字段：

- `host: String`
- `port: u16`（默认 22）
- `user: String`（允许缺失，但连接前必须补齐）
- `source: DraftSource`（`DirectInput` / `FromSaved(session_id)` / `FromRecent(recent_id)`）

### 2) 已保存会话（Saved Session / SessionProfile）

可在设置中心连接列表中展示与编辑的会话配置（含非敏感字段）。

与当前实现对齐（参考 `SSH连接与凭据安全链路分析.md`）：

- `SessionProfile.transport = TransportConfig::Ssh(SshConfig { host, port, user, auth, credential_id })`
- 非敏感字段落盘：`sessions.json`

### 3) 凭据（Credential / Vault Payload）

敏感字段存储在 Vault（`vault.json`）中，并通过 `credential_id` 关联到会话。

与当前实现对齐：

- `credential_id` 建议规则继续使用 `ssh:{session_id}` 或同等可推导/可稳定的键
- 载荷：`SshCredentialPayload { password: Option<String>, passphrase: Option<String> }`

## Launcher 页面结构（信息架构）

Launcher 面板由三块组成：

1. 顶部：**搜索/输入一体框**
2. 中部：**最近会话列表**
3. 底部：**已保存会话列表（按组分组）**

### 顶部输入框行为（两种模式共存）

- **搜索模式**：仅搜索“已保存会话列表”
  - 支持匹配：会话名、host/IP（含域名）、可选 user
- **直连模式**：输入并解析 `user@host[:port]`（以及 `host[:port]`、`host`）

输入框提示文案建议：

- `搜索已保存会话，或输入 user@host[:port]`

---

## 输入解析与模式判定

### 直连判定（满足任一即认为“直连候选”）

- 输入包含 `@`（如 `user@host`）
- 或输入“看起来像 host”：
  - 域名/IPv4（含 `.`）
  - IPv6（建议支持 `user@[fe80::1]:2222` 形式）
  - 明确包含 `:port` 的形态

### 支持的直连输入格式

- `user@host`
- `user@host:port`
- `host:port`
- `host`
- `user@[ipv6]:port`

解析输出：

- `user: Option<String>`
- `host: String`
- `port: Option<u16>`

默认值：

- `port` 缺省为 22

连接前约束：

- `host` 必填
- `user` 必填（若缺失则进入“补齐用户名”弹窗/小面板）

## 认证策略（默认优先自动探测）

### 认证尝试顺序（默认）

1. **SSH Agent**
2. **本地 Key**
3. **Keyboard-Interactive**（若服务器要求/返回挑战）
4. **Password**（兜底）

说明：

- 该顺序体现“默认优先自动探测（agent/key）”的决策。
- UI 应支持在失败后“切换认证方式”重试（不丢 Draft / 不丢会话上下文）。

### 首次使用自动探测：授权提醒（一次性）

触发条件：

- 用户第一次进入“自动探测（agent/key）”路径（任一尝试前弹出/或第一次成功前弹出，二选一，但需一致）。

提醒要点：

- 将使用系统 SSH agent 或本机密钥来完成认证
- 不会上传私钥，仅在本机使用

按钮：

- `允许（本次）`
- `始终允许`
- `改用密码`

持久化：

- 将“始终允许”存入设置（建议 `settings.security` 或 `settings.connection` 相关区域）

## 直连成功必保存（策略 A：强一致）与去重规则

直连（从输入框发起）在**连接成功后**必须固化为“已保存会话 + 凭据”。

### 去重键（用于避免列表膨胀）

建议逻辑去重键：


K = (user, host, port)


行为：

- 若已存在同 `K` 的 Saved Session：
  - 复用该会话记录（不创建新会话）
  - 更新 `last_connected_at`（若有）
  - 仅在必要时更新可覆盖字段（例如：若此前缺失 `credential_id`，则补上；避免覆盖用户自定义名称/分组）
- 若不存在：
  - 创建新 `SessionProfile`，并放入默认分组（如“未分组/最近导入”）

### 保存时机

- 必须在“连接成功打开 Tab”之后立即触发固化（避免保存一堆失败草稿）。

### 凭据保存规则

- 若本次连接过程中用户输入了密码/交互式响应，且连接成功：
  - 将密码写入 Vault，并将 `credential_id` 绑定到该会话
- 若使用 agent/key 成功：
  - 不强制创建 password 凭据；可记录“上次成功认证方式”（非敏感）用于下次优化顺序

## 从最近/已保存会话点击连接

### 情况 1：已有认证信息（密码/密钥等）

- 直接连接（进入进度态）
- 若失败归因为“凭据失效/缺失”：
  - 进入“认证修复”弹窗（允许改用密码/重新选 key/切换 agent）

### 情况 2：没有认证信息

- 默认弹出**密码输入框**
- 输入后连接
- **连接成功后保存密码**到 Vault，并绑定会话（写入 `credential_id`）

---

## 密码错误多次：中断连接流程

需求：密码错误在多次错误时中断连接流程。

### 推荐阈值

- 同一次连接尝试中，连续密码错误 N 次（建议 3）后中断。

### 中断后的 UX

- 明确提示：`密码多次错误，已中断本次连接`
- 保留上下文（Draft/会话不丢）
- 提供动作：
  - `返回编辑`
  - `稍后重试`
  - `切换认证方式`（若 agent/key 可用）

### 工程约束

- 中断应停止后续认证尝试（避免密码锁定策略触发、避免无限循环）
- 中断不应自动保存任何凭据

## 状态机（State / Event / Action）

本节给出可直接映射到 reducer / message loop 的状态机定义（简化版本，后续可扩展）。

### 状态（State）

- `Idle`：Launcher 打开但未发起连接
- `Typing { input }`：输入变化中（可同时维护 search results / parse results）
- `NeedUser { host, port }`：直连解析到 host 但缺 user，需要补齐
- `Connecting { draft, attempt, progress }`：
  - `attempt`：当前认证尝试类型（Agent/Key/Interactive/Password）
  - `progress`：解析/握手/认证/打开会话
- `NeedAuthPassword { draft, error_count }`：需要用户输入密码（或密码错误重试）
- `NeedAuthInteractive { draft, challenge_state }`：keyboard-interactive 多步挑战
- `AuthLocked { draft }`：密码错误过多，中断
- `Connected { session_tab_id, saved_session_id }`：已连接并（若直连）已固化
- `Failed { draft, reason }`：失败态（可重试/切换方式/编辑）

### 事件（Event）

- `InputChanged(text)`
- `SubmitConnect`
- `PickSavedSession(session_id)`
- `PickRecentSession(recent_id)`
- `CancelConnect`
- `AuthPolicyConsent(choice)`（首次自动探测授权）
- `PasswordEntered(pw)`
- `InteractiveResponded(resp)`
- `ConnectProgressUpdated(stage)`
- `ConnectSucceeded(runtime_session)`
- `ConnectFailed(error)`
- `SwitchAuthMethod(method)`

### 动作（Action）

- `ParseInput`（生成 parse result + mode）
- `SearchSavedSessions(query)`（仅 saved sessions）
- `BuildDraftFromInput/FromSaved/FromRecent`
- `StartConnect(draft, preferred_auth_order)`
- `LoadCredentialIfAny(session_id/credential_id)`
- `PersistOnSuccess`：
  - `UpsertSessionProfile`（策略 A：直连成功必保存，且按去重键合并）
  - `SaveCredentialToVault`（仅在有用户输入且成功后）
- `OpenSessionTab(runtime_session)`
- `StopConnect`（中断/取消）

## 错误态归因与可操作提示（最小集合）

只要求实现“能驱动下一步”的最小归因集合，避免把底层错误原样抛 UI。

- `AuthFailed`：
  - 动作：重输密码 / 切换认证方式
- `HostUnreachable`（DNS/网络不可达）：
  - 动作：编辑 host/port / 检查网络 / 代理（若支持）
- `HostKeyChanged/Unknown`（若后续接入 HostKeyPolicy）：
  - 动作：查看指纹 / 接受或拒绝（Ask/AcceptNew）
- `KeyNotFound/PermissionDenied`（key 路径不可读）：
  - 动作：选择其他 key / 改用密码

## 与现有实现对齐点（代码层映射提示）

本流程在代码层建议复用/扩展以下结构（名称以现有文档为准）：

- `SessionProfile` / `SshConfig`：保存非敏感字段到 `sessions.json`
- `credential_id` + `src/vault/session_credentials.rs`：保存/读取敏感字段
- UI 侧“连接弹窗/连接提示态”：需要支持从“列表点击”与“输入直连”两种来源生成 Draft

注意现状差异（来自 `SSH连接与凭据安全链路分析.md`）：

- 当前实际只接通了 password 认证；agent/key/interactive 在模型层有定义但未接入连接实现。
- 因此本流程文档可以作为“打通认证方式”的产品与工程目标定义。

## 验收标准（Acceptance Criteria）

### 直连与保存（策略 A）

- 输入 `user@host:port` 成功连接后：
  - 自动创建或合并一条 Saved Session（按去重键）
  - 设置中心连接列表可见且可编辑
- 同 `user/host/port` 再次直连成功：
  - 不新增重复条目（只更新最近连接时间等元数据）

### 自动探测授权

- 首次触发 agent/key 自动探测时：
  - 出现一次性授权提醒
  - 选择“改用密码”可顺利走密码连接

### 点击会话直连与凭据补齐

- 点击已有凭据会话：不弹框直接连接（成功打开 Tab）
- 点击无凭据会话：弹出密码框；成功后保存密码，下次点击直接连

### 密码错误中断

- 同一次连接尝试中连续输错密码达到阈值：
  - 连接流程中断（不继续尝试）
  - 保留上下文并提供返回/重试/切换方式

建议埋点（不要求一次性全做，但应预留事件名）：

- `launcher.open`

- `launcher.direct_input.detected`
- `connect.start`（来源：direct/saved/recent）
- `auth.attempt`（agent/key/interactive/password）
- `auth.consent.shown` / `auth.consent.choice`
- `connect.success` / `connect.fail`（归因码）
- `credential.save.success/fail`
- `session.upsert.success/fail`
- `auth.password.error_count`（仅计数）
