🎯 RustSSH 组件设计规范（Design Spec v1.0）

核心原则：层级驱动 + 材质表达 + 极简反馈（非叠加色）

⸻

1.4 动效（必须统一）

transition: all 120ms cubic-bezier(0.4,0,0.2,1);

点击：

transform: scale(0.97)
duration: 80ms

缓动函数选用规则：

场景 缓动函数
按钮悬停进入 ease_out
按钮按下 ease_in
按钮悬停退出 ease_in
Tab 切换 ease_in_out
模态打开 ease_out
模态关闭 ease_in
颜色过渡 ease_out
加载/进度 linear

时长分级（引用 layout.rs 常量，禁止硬编码）：

档位 时长 适用场景
快速 60-80ms 按钮按下缩放
标准 100-150ms 悬停过渡、Tab 切换
从容 200-300ms 模态弹窗打开/关闭
即时 0ms 纯状态变化

详细规范见 `doc/动画使用规范.md`。

5.4 动效统一

类型 时间
Hover 120ms
Click 80ms
Tab 切换 150ms

性能红线（终端渲染零动画原则）：

终端正文渲染路径（terminal_rich.rs styled_terminal）严禁添加任何动画。
RowWidgetCache 优化依赖于 generation-based cache hit，改变样式属性会导致整个缓存失效。
唯一例外：光标 alpha sin 插值。

缓动函数引用 src/theme/animation.rs，禁止硬编码时长数字（引用 layout.rs 常量）。
超过 300ms 的交互反馈动画会使用户感觉迟钝，禁止使用。

详细规范见 `doc/动画使用规范.md`。
