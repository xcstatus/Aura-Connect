# RustSSH 弹窗（Modal）使用规范 v1.0

> 本规范是 RustSsh UI 开发的核心规范之一，与 `doc/组件设计规范.md`、`doc/动画使用规范.md` 配套使用。

---

## 一、弹窗分类

| 类型 | 名称 | 用途 | 层级 | 示例 |
|------|------|------|------|------|
| **Terminal-inline Overlay** | 终端内嵌浮层 | 连接进度、凭证输入 | Z-1 ~ Z-2 | `inline_connecting_overlay`, `inline_password_overlay` |
| **Primary Modal** | 主弹窗 | 设置中心、会话编辑器 | Z-3 ~ Z-6 | `settings_modal`, `session_editor_modal` |
| **Secondary Modal** | 子弹窗 | 依赖主弹窗存在 | Z-4 ~ Z-6 | `vault_modal`, `auto_probe_consent_modal` |
| **System Modal** | 系统弹窗 | 必须响应，无法绕过 | Z-7 | `host_key_prompt_modal`, `vault_unlock_modal` |
| **Debug Overlay** | 调试覆盖层 | 开发调试用 | Z-8 | `debug_overlay` |

## 二、渲染架构

### 2.1 Stack 分层顺序

所有弹窗通过 `Stack` 叠加渲染，层级由代码中的添加顺序决定：

```rust
// view.rs - 层级从底到顶
let mut layers = vec![
    below_top_fill,                    // 0. 主页（终端 + 顶栏 + 底栏）
    inline_connecting_overlay,          // 1. 连接进度条
    inline_password_overlay,            // 2. 密码输入浮层
];
// settings_anim 分支
layers.push(settings_modal);           // 3. 设置中心（Primary）
layers.push(session_editor_modal);     // 4. 会话编辑器（Secondary，依赖 Settings）
layers.push(vault_modal);              // 5. 密码库（Secondary，依赖 Settings）
layers.push(auto_probe_consent_modal);// 6. 自动探测确认（Secondary，依赖 Settings）
layers.push(host_key_prompt_modal);    // 7. 主机密钥（System，始终在 Settings 上）
layers.push(vault_unlock_modal);      // 8. 密码库解锁（System，最高优先级）
if debug_overlay_enabled {
    layers.push(debug_overlay);        // 9. 调试覆盖（最顶层）
}
```

### 2.2 渲染函数签名规范

每个弹窗必须遵循以下签名：

```rust
pub(crate) fn xxx_modal(state: &IcedState) -> Element<'_, Message> {
    // 1. 可见性检查：不可见时返回 Space
    let Some(data) = state.xxx_data.as_ref() else {
        return Space::new().into();
    };

    // 2. 获取 DesignTokens
    let tokens = tokens_for_state(state);

    // 3. 构建遮罩 + 卡片 + Stack
    // ...
}
```

## 三、遮罩层（Scrim）规范

### 3.1 Scrim 类型选用

| 场景 | Scrim 函数 | 透明度 |
|------|-----------|--------|
| 独立弹窗（无嵌套） | `layered_scrim_style(tokens, 0)` | 35% |
| 主弹窗（层级 0） | `layered_scrim_style(tokens, 0)` | 35% |
| 嵌套弹窗（层级 1+） | `layered_scrim_style(tokens, level)` | +15% per level，上限 75% |
| 动画过渡中 | `layered_scrim_alpha(alpha, level)` | 动态计算 |

### 3.2 Scrim 透明度递增规则

```rust
// tokens.rs - layered_scrim
base_alpha = 0.35
per_level = 0.15
max_alpha = 0.75
effective_alpha = min(base_alpha + level * per_level, max_alpha)
```

### 3.3 遮罩行为规范

**当前设计**：遮罩**不拦截**点击事件，用户可以点击主页面（被半透明遮挡）。

**后续改进方向**（如需严格模态）：
- 在 scrim 的 `container` 上添加 `.on_press(DismissMessage)` 实现点击外部关闭
- ESC 键在 `subscription.rs` 中统一处理

## 四、动画规范

### 4.1 动画状态机

```rust
pub(crate) enum ModalAnimPhase {
    Closed,    // 隐藏
    Opening,   // 打开动画中
    Open,      // 完全打开
    Closing,   // 关闭动画中
}
```

### 4.2 动画时长常量

所有时长必须引用 `layout.rs` 中的常量，禁止硬编码：

```rust
// layout.rs
DURATION_MODAL_MS: 200        // 打开动画
DURATION_MODAL_CLOSE_MS: 150 // 关闭动画
```

### 4.3 动画插值函数

在 `state.rs` 中实现以下方法：

```rust
impl IcedState {
    // Alpha 插值（0.0 → 1.0）
    pub(crate) fn quick_connect_anim_alpha(&self, tick_ms: f32) -> f32 { ... }

    // Y 偏移插值（-8.0 → 0.0）
    pub(crate) fn quick_connect_anim_offset(&self, tick_ms: f32) -> f32 { ... }

    // 动画完成检查
    fn advance_modal_animations(&mut self) { ... }
}
```

### 4.4 动画期间渲染策略

```rust
// 关闭动画期间仍需渲染弹窗，以确保动画完整播放
if state.quick_connect_anim.phase != ModalAnimPhase::Closed {
    layers.push(quick_connect_modal_stack(state));
}
```

缓动函数选用规则（详细见 `doc/动画使用规范.md`）：

| 动画场景 | 缓动函数 | 时长 |
|----------|----------|------|
| 模态打开 | ease_out | DURATION_MODAL_MS (200ms) |
| 模态关闭 | ease_in | DURATION_MODAL_CLOSE_MS (150ms) |
| 颜色过渡 | ease_out | 120ms |

## 六、事件处理规范

### 6.1 关闭触发

| 触发方式 | 实现位置 | 说明 |
|---------|---------|------|
| 点击关闭按钮 | 各弹窗 `icon_close_button()` | 必需 |
| ESC 键 | `subscription.rs` | 统一处理 |
| 点击遮罩外部 | **当前不实现** | 注释明确禁用 |

### 6.2 关闭按钮规范

```rust
fn icon_close_button(tokens: DesignTokens, close_msg: Message) -> Element<'static, Message> {
    let close_icon = icon_view_with(
        IconOptions::new(IconId::Close)
            .with_size(14)
            .with_color(tokens.text_secondary),
        close_msg,
    );
    button(close_icon)
        .on_press(close_msg)
        .width(iced::Length::Fixed(28.0))
        .height(iced::Length::Fixed(28.0))
        .style(style_top_icon(tokens))
        .into()
}
```

### 6.3 按钮样式规范

| 按钮类型 | 样式函数 | 用途 |
|---------|---------|------|
| Primary | `style_chrome_primary(tokens)` | 主要操作（确认、连接） |
| Secondary | `style_chrome_secondary(tokens)` | 次要操作（取消、返回） |
| Icon | `style_top_icon(tokens)` | 工具栏图标按钮 |

## 七、多级弹窗规则

### 7.1 层级关系

```
Settings (Z-3)
    ├── SessionEditor (Z-4, 依赖 Settings)
    ├── VaultModal (Z-5, 依赖 Settings)
    └── AutoProbeConsent (Z-6, 依赖 Settings)

HostKeyPrompt (Z-7, 独立于 Settings)
VaultUnlock (Z-8, 最高优先级)
```

### 7.2 嵌套弹窗的 Scrim 层级

```rust
// Settings 分支内：所有嵌套弹窗共享 level=0
if state.settings_anim.phase != Closed {
    layers.push(settings_modal);              // scrim level=0, alpha=0.35
    layers.push(session_editor_modal(state)); // scrim level=0（当前实现）
    // 建议：session_editor_modal 使用 level=1，alpha=0.50
    layers.push(vault_modal(state));
    layers.push(auto_probe_consent_modal(state));
}
```

### 7.3 业务规则

1. **System Modal 必须置顶**：`vault_unlock` 和 `host_key_prompt` 不在任何条件分支内，始终渲染
2. **Secondary Modal 依赖 Primary**：只有当 Settings 打开时才渲染嵌套弹窗
3. **避免深度嵌套**：超过 2 层嵌套时考虑合并或重构

## 九、当前实现问题与改进建议

| 问题 | 现状 | 建议 |
|------|------|------|
| 主页面可点击 | 遮罩不拦截事件 | 如需严格模态，添加 scrim 的 `on_press` |
| 嵌套弹窗 Scrim 层级 | 均为 level=0 | 嵌套弹窗使用 level=1，alpha 递增 |
| ESC 关闭 | 注释说"在其他地方处理" | 确认 `subscription.rs` 实现完整性 |
| 尺寸硬编码 | 部分弹窗直接写 520px | 统一引用 `layout.rs` 常量 |
| Focus 管理 | 无焦点管理 | 考虑添加 `focus_group_id` |

## 十、禁止行为

1. **禁止硬编码动画时长**：必须引用 `layout.rs` 常量
2. **禁止硬编码颜色**：必须通过 `DesignTokens` 获取
3. **禁止硬编码尺寸**：Settings 宽度禁止写死 800，必须用比例计算
4. **禁止修改 Stack 层级顺序**：Z-7/Z-8 固定位置，System Modal 必须最高
5. **禁止在动画计算中创建新对象**：使用 `f32` 和 `iced::Color` 直接运算
6. **禁止超过 300ms 的交互反馈动画**
