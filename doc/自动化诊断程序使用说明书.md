# RustSSH 自动化诊断程序使用说明书

## 概述

RustSSH 提供了一套完整的自动化诊断程序，用于终端渲染调试、性能基线采集、键盘回归验证和 GPU 渲染回归检测。本说明书详细介绍各脚本的用途、命令行参数及使用流程。

---

## 一、终端诊断开关

### 1.1 总开关

| 环境变量 | 说明 |
|---------|------|
| `RUST_SSH_TERM_DIAG=1` | 开启所有终端诊断日志 |

### 1.2 分项开关

| 环境变量 | 说明 | 对应模块 |
|---------|------|---------|
| `RUST_SSH_TERM_DIAG_HIT_TEST=1` | 单元格命中测试日志 | `term_hit_test` |
| `RUST_SSH_TERM_DIAG_VIEWPORT=1` | 视口几何信息日志 | `term_viewport` |

### 1.3 使用方式

通过环境变量开启，配合 `RUST_LOG` 指定模块日志级别：

```bash
# 开启命中测试诊断
RUST_SSH_TERM_DIAG_HIT_TEST=1 RUST_LOG=term_hit_test=debug cargo run

# 开启视口几何诊断
RUST_SSH_TERM_DIAG_VIEWPORT=1 RUST_LOG=term_viewport=debug cargo run

# 开启全部诊断
RUST_SSH_TERM_DIAG=1 cargo run
```

> **说明**：`RUST_SSH_PERF_DUMP` 等性能采集开关保持不变，诊断开关仅控制终端调试日志是否输出。

## 二、键盘回归矩阵

### 2.1 脚本信息

- **脚本路径**：`scripts/keyboard_matrix.sh`
- **对应测试**：`tests/keyboard_matrix.rs`
- **用途**：验证键盘编码不变量，适用于 `vim`/`tmux`/`top` 类场景的基础保障

### 2.2 使用命令

```bash
./scripts/keyboard_matrix.sh
```

### 2.3 功能说明

该脚本执行键盘编码自动化回归测试，确保终端对各类键盘事件的编码结果在版本迭代中保持一致，是 P0/DoD（交付标准）级别的回归保障。

## 三、性能基线采集

### 3.1 `perf_baseline.sh`

#### 基本信息

- **用途**：对已运行的 RustSsh 进程按秒采样，输出 CPU 使用率和内存占用 CSV
- **输出格式**：`ts_ms,cpu_pct,rss_kb`

#### 命令行参数

| 参数 | 必填 | 说明 |
|-----|------|------|
| `--pid <PID>` | 是 | RustSsh 进程 PID |
| `--seconds <N>` | 否 | 采样总时长（秒），默认 120 |
| `--out <file>` | 否 | 输出 CSV 文件路径 |

#### 使用示例

```bash
# 采样 120 秒，输出到 term-dumps/idle_120s.csv
./scripts/perf_baseline.sh --pid 12345 --seconds 120 --out term-dumps/idle_120s.csv

# 采样 60 秒（使用默认路径）
./scripts/perf_baseline.sh --pid 12345 --seconds 60
```

### 3.2 `p2_1_collect.sh`

#### 基本信息

- **用途**：P2.1 "一键采集"脚本，自动生成三段性能采样 CSV，并打印每段统计摘要（均值/峰值/方差）
- **输出目录**：`term-dumps/`（可通过参数修改）

#### 生成产物

| 文件名 | 说明 | 默认时长 |
|-------|------|---------|
| `p2_1_meta.json` | 版本/设置/采集参数元数据 | — |
| `p2_1_idle_120s.csv` | 空闲状态采样 | 120 秒 |
| `p2_1_throughput_180s.csv` | 吞吐量测试采样 | 180 秒 |
| `p2_1_longrun_1800s.csv` | 长时间运行采样 | 1800 秒 |

#### 命令行参数

| 参数 | 必填 | 说明 |
|-----|------|------|
| `--pid <PID>` | 是 | RustSsh 进程 PID |
| `--idle-seconds <N>` | 否 | 空闲采样时长（默认 120） |
| `--throughput-seconds <N>` | 否 | 吞吐量采样时长（默认 180） |
| `--longrun-seconds <N>` | 否 | 长时间运行采样时长（默认 1800） |

#### 使用示例

```bash
# 标准采集
./scripts/p2_1_collect.sh --pid 12345

# 缩短试跑（仅验证流程）
./scripts/p2_1_collect.sh --pid 12345 --idle-seconds 30 --throughput-seconds 60 --longrun-seconds 120
```

#### 各阶段说明

1. **Idle 阶段**：应用处于空闲状态，采集基线资源占用数据
2. **Throughput 阶段**：需要在应用内执行高输出命令（如 `seq 1 200000`），采集大流量数据处理能力
3. **Longrun 阶段**：长时间稳定性测试，需保持应用运行（可选间歇 burst）

## 四、GPU 渲染回归

### 4.1 脚本信息

- **脚本路径**：`scripts/terminal_gpu_regression.sh`
- **用途**：跑 GPU 终端渲染回归矩阵，输出 compare/offscreen 产物到 `term-dumps/`
- **输出目录**：可通过 `RUST_SSH_TERM_DUMP_DIR` 环境变量修改

### 4.2 使用命令

```bash
# 使用默认输出目录
./scripts/terminal_gpu_regression.sh

# 指定输出目录
RUST_SSH_TERM_DUMP_DIR=/path/to/output ./scripts/terminal_gpu_regression.sh
```

### 4.3 功能说明

该脚本执行 GPU 终端渲染的自动化回归测试，通过对比产物（compare）和离屏渲染结果（offscreen）验证渲染输出在版本迭代中的一致性。

## 五、应用内性能诊断

### 5.1 Debug Overlay（性能覆盖层）

RustSSH 内置了实时性能诊断覆盖层，用于在运行时观察终端渲染性能指标。

#### 开启方式

按 `Ctrl+Shift+D` 切换显示/隐藏 Debug Overlay

#### 显示内容

Debug Overlay 展示以下实时指标：

| 指标 | 说明 |
|------|------|
| `tick` | 每秒 tick 次数 |
| `pump` | 每秒 PTY 数据泵送次数 |
| `bytes` | 每秒输入字节数 |
| `slow` | 慢帧占比（>16ms） |
| `empty` | 空读取占比 |
| `vt` | VT 解析耗时 |
| `frame` | 帧渲染耗时 |
| `tick-avg` | 单次 tick 平均耗时 |
| `tick-max` | 单次 tick 最大耗时 |
| `focus` | 窗口焦点状态 |
| `tabs` | 当前标签页数量 |

#### Per-tab 指标

每个标签页独立显示：

```
[idx] pump=N bytes=N vt=Nms frame=Nms
```

#### Tick 历史

显示最近一段时间的 tick 耗时直方图（毫秒）。

#### 颜色编码

| 颜色 | 含义 |
|------|------|
| 绿色 | 性能正常（slow < 5%） |
| 黄色 | 性能警告（slow 5-10% 或 empty > 50%） |
| 红色 | 性能异常（slow > 10%） |

### 5.2 性能数据导出

```bash
RUST_SSH_PERF_DUMP=term-dumps/perf.csv cargo run
```

应用退出时将性能数据导出到指定 CSV 文件，适用于无法使用外部 `ps` 命令的环境。

## 六、终端诊断代码 API

### 6.1 诊断开关函数

在 `src/term_diag.rs` 中提供了统一的诊断开关检查接口：

```rust
// 检查总开关或指定分项开关
if crate::term_diag::enabled("HIT_TEST") {
    // 输出命中测试诊断日志
}

// 检查总开关（所有诊断）
if crate::term_diag::enabled("") {
    // 输出通用诊断日志
}
```

### 6.2 支持的诊断范围

- `HIT_TEST`：单元格命中测试
- `VIEWPORT`：视口几何信息
- `VT_LOG_FRAME`：帧日志（通过 `RUST_SSH_VT_LOG_FRAME` 控制）

## 七、使用流程建议

### 7.1 开发调试流程

1. **日常开发**：使用 `Ctrl+Shift+D` 观察 Debug Overlay，确保帧率正常
2. **问题定位**：根据问题类型开启对应诊断开关，查看详细日志
3. **回归验证**：修改代码后运行 `keyboard_matrix.sh` 确保键盘编码无误

### 7.2 性能基线采集流程

1. 启动 RustSsh 应用
2. 运行 `p2_1_collect.sh --pid <APP_PID>` 开始采集
3. 按提示在各阶段执行相应操作
4. 采集完成后查看 `term-dumps/` 目录下的统计摘要

### 7.3 发布前验证流程

1. 运行 `keyboard_matrix.sh`（P0 回归）
2. 运行 `terminal_gpu_regression.sh`（GPU 渲染回归）
3. 采集性能基线并与历史数据对比
4. 检查 Debug Overlay 各指标是否在正常范围

## 九、故障排查

### 9.1 采样脚本无输出

- 确认进程 PID 正确
- 确认进程有 CPU 和内存访问权限
- 检查 `ps` 命令是否可用

### 9.2 Debug Overlay 显示异常数据

- slow 持续 > 10%：检查是否有后台进程占用 CPU
- empty 持续 > 50%：确认 SSH 连接正常、数据流正常
- 帧率异常：检查 GPU 驱动是否正常

### 9.3 键盘回归失败

- 检查测试用例与实际键盘布局是否匹配
- 确认测试环境与开发环境键盘配置一致
