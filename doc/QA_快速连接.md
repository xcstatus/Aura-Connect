## QA 手测脚本：快速连接（Launcher）

本手测脚本覆盖文档《快速连接入口：流程与状态机》主要分支，供 QA/产品验收使用。

### 0. 前置条件
- 至少准备 1 台可登录的 SSH 主机 A（密码认证可用）
- （可选）准备 1 台要求 keyboard-interactive 的主机 B（或在 A 上开启 PAM/2FA 等挑战）
- （可选）准备一把可用私钥（key 登录）
- （可选）系统已启动 SSH Agent 且已加载至少 1 个 key（`ssh-add -l` 有输出）
- （可选）Vault 已初始化（安全中心设置主密码）

### 1. 打开入口与搜索/直连模式
- 打开快速连接面板
  - 期望：显示「最近」与「已保存」列表
- 在顶部输入框输入普通关键词（例如会话名的一部分）
  - 期望：仅过滤「已保存」会话列表
- 在顶部输入框输入 `user@host:22`
  - 期望：出现「连接到 …」主按钮（CTA）
  - 点击 CTA
  - 期望：进入表单，Host/Port/User 已自动填入

### 2. user 缺失补齐（NeedUser）
- 在顶部输入框输入 `host:22`（不带 user）
  - 点击 CTA
  - 期望：进入表单，提示需要补齐 user；补齐后可继续连接

### 3. Password 连接与错误中断（AuthLocked）
- 选择认证方式 Password，输入错误密码，连续点击连接 3 次
  - 期望：第 3 次后进入「中断/锁定」态，本次连接不再继续尝试
- 修改密码输入或切换认证方式
  - 期望：错误计数重置，可再次连接

### 4. Saved/Recent 点击分支
- 从「已保存」列表点击一个会话
  - 若有凭据：期望自动连接成功
  - 若无凭据：期望进入表单提示补齐密码后连接
- 连接成功后回到快速连接面板
  - 期望：最近列表出现该记录

### 5. 策略 A：直连成功必保存 + 去重（K=user,host,port）
- 使用顶部输入框直连到 A 并成功
  - 期望：会话被固化为「已保存」记录
- 再次用相同 `user/host/port` 直连成功
  - 期望：不新增重复条目（复用原条目）

### 6. Key 认证（含 passphrase）
- 选择 Key，填写私钥路径（与可选 passphrase），连接
  - 期望：连接成功
- 填写一个不可读/不存在路径
  - 期望：归因提示为 KeyNotFound 或 PermissionDenied，可切换到 Password

### 7. Agent 认证
- 选择 Agent，连接
  - 期望：连接成功（若 agent 可用且含可用 key）
  - 若 agent 不可用：期望提示 AgentUnavailable，可切换认证方式

### 8. Keyboard-Interactive 多步挑战
- 选择 Interactive，点击连接
  - 期望：若服务端发 challenge，UI 展示 prompts 输入框
  - 逐步填写并提交（可能多轮）
  - 期望：最终连接成功或失败归因明确

### 9. Vault：连接成功后引导解锁保存凭据
- Vault 已初始化但当前运行时未解锁
- 使用 Password 连接成功
  - 期望：弹出「解锁 Vault 以保存凭据」提示；取消则仅本次连接不保存
  - 输入主密码确认
  - 期望：凭据保存成功；下次点击该会话可自动回填/自动连接
