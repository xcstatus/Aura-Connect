# RustSSH 设计系统：全场景动态色卡 (Color Palette)

此文档定义了 RustSSH 客户端在不同主题下的核心视觉标准，旨在保证跨平台 UI 的一致性与高级审美。

## 3. 使用规范 (Usage Guidelines)

1.  **层级优先 (Z-Index Logic)**: 
    *   背景层 (`Bg_Primary`) 始终作为底层。
    *   交互层 (`Bg_Header`, `Popups`) 必须使用略亮于（暗模式下）或略暗于（亮模式下）主背景的颜色。
2.  **对比度校验 (AA/AAA Standard)**: 
    *   核心代码区必须满足 **4.5:1** 以上的对比度。
    *   处于非活跃状态的文字 (`Text_Secondary`) 允许降至 3:1。
3.  **科技感加持 (Glow/Bloom)**: 
    *   对于 `Accent_Base` (如选中横条)，建议开启微弱的 Bloom 模糊效果（Alpha 30%-50% 的同色阴影）。
4.  **图标一致性**: 
    *   图标 (`Icon`) 的颜色应跟随对应的 `Text_Primary` 调色板。悬浮时增加 10% 的 Alpha 亮度加成。
